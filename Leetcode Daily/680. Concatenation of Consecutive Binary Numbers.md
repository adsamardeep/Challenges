# 🔢 Concatenation of Consecutive Binary Numbers

## 📜 Problem Statement

Given an integer `n`, form a binary string by concatenating the binary representations of all integers from `1` to `n` in order.

Return the decimal value of this binary string modulo:

```

10^9 + 7

```

---

## 🔍 Example

### Example 1:
**Input:**
```

n = 3

```

**Binary concatenation:**
```

1 → "1"
2 → "10"
3 → "11"

Concatenated → "11011"

```

**Decimal value:**
```

27

```

**Output:**
```

27

```

---

### Example 2:
**Input:**
```

n = 12

```

**Output:**
```

505379714

```

---

## 🧠 Key Insight

Instead of actually building a huge string (which would be slow and memory-heavy),  
we use **bit shifting**.

When we append a binary number `x`:

- Determine its bit width:  
```

bw = x.bit_length()

````
- Shift the current result left by `bw` bits
- Add `x`

This simulates binary concatenation efficiently.

---

## 🚀 Step-by-Step Explanation

For each number `x` from `1` to `n`:

```python
bw = x.bit_length()
ans = (ans << bw) + x
````

Why this works:

If:

```
ans = 110 (binary)
x = 11 (binary, 2 bits)
```

Then:

```
ans << 2 = 11000
+ x      = 11011
```

Exactly like concatenation.

---

## 🔒 Modulo Handling

Since numbers grow very large:

```python
if ans >= MOD:
    ans %= MOD
```

We reduce using modulo `10^9 + 7`.

---

## 💻 Code 

```python
class Solution:
    def concatenatedBinary(self, n: int) -> int:
        MOD = 10**9 + 7
        ans = 0

        for x in range(1, n + 1):
            bw = x.bit_length()
            ans = (ans << bw) + x
            if ans >= MOD:
                ans %= MOD

        return ans
```

---

## ⏱ Complexity Analysis

Let `n` be the input.

### Time Complexity:

* Loop runs `n` times
* Each iteration is O(1)
* **O(n)**

### Space Complexity:

* Only integer variables
* **O(1)**

---

## 🧪 Edge Cases

1. `n = 1` → 1
2. `n = 2` → "110" → 6
3. Large `n` (e.g., 10⁵) → handled efficiently
4. Bit-length increases at powers of 2

---

## 🎯 Why This Is Optimal

* No string building
* No repeated binary conversions
* Pure bit manipulation
* Efficient modulo handling
