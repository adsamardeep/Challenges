## 🏷 Get the Three Largest Distinct Rhombus Border Sums in a Grid

## 📜 Problem Statement
Given an `m x n` grid of integers, a **rhombus** is defined by selecting a center cell and extending equally in four diagonal directions. The **rhombus sum** is the sum of the values along the border of that rhombus.  
Return the **three largest distinct rhombus border sums** in descending order.

## 🔍 Example Input/Output

**Example**

Input:
```
grid = [
 [3,4,5,1,3],
 [3,3,4,2,3],
 [20,30,200,40,10],
 [1,5,5,4,1],
 [4,3,2,2,5]
]
```

Output:
```
[228, 216, 211]
```

Explanation:
Different rhombus shapes produce different border sums. We return the **three largest distinct sums**.

---

## 🧠 Approach

### 1. Observations
A rhombus border consists of **four diagonal edges**:
- top → right
- right → bottom
- bottom → left
- left → top

To compute these efficiently, we precompute **diagonal prefix sums**.

### 2. Prefix Sum Structures

Two arrays are maintained:

1. `diag`
   - Stores prefix sums along the **main diagonals** (`\` direction).

2. `antid`
   - Stores prefix sums along the **anti-diagonals** (`/` direction).

An offset is used so negative indices from `(i - j)` become valid.

### 3. Computing Rhombus Border Sum

For a rhombus centered at `(i, j)` with radius `d`:

```
Top    = (i-d, j)
Right  = (i, j+d)
Bottom = (i+d, j)
Left   = (i, j-d)
```

The border is composed of four diagonal segments which are retrieved in **O(1)** using prefix sums.

Special case:
```
d = 0 → rhombus is a single cell
```

### 4. Enumerate All Possible Rhombuses

Maximum radius:
```
d ≤ min(m, n) // 2
```

For each `(i, j)` center and radius `d`:
- Compute rhombus border sum.
- Maintain the **top three distinct sums**.

### 5. Maintain Top 3 Distinct Values

Use a small fixed array `x = [-1, -1, -1]` and update it whenever a larger distinct value appears.

### 6. Final Result

Remove `-1` placeholders and return the remaining values in descending order.

---

## 💻 Code 

```python
diag=[[0]*51 for _ in range(100)]
antid=[[0]*51 for _ in range(100)]
OFFSET=50
class Solution:
    def getBiggestThree(self, grid: List[List[int]]) -> List[int]:
        def rhombusSum(i, j, d):
            if d==0: return grid[i][j]
            l, r, u, b=j-d, j+d, i-d, i+d
            i0, i1=u-j+OFFSET, i-l+OFFSET
            Sum=diag[i0][r+1]-diag[i0][j]
            Sum+=diag[i1][j+1]-diag[i1][l]

            j0, j1=u+j, b+j
            Sum+=antid[j0][i]-antid[j0][u+1]
            Sum+=antid[j1][b]-antid[j1][i+1]
            return Sum

        m, n=len(grid), len(grid[0])
        for i, row in enumerate(grid):
            for j, x in enumerate(row):
                i0, j0=i-j+OFFSET, i+j
                diag[i0][j+1]=diag[i0][j]+x
                antid[j0][i+1]=antid[j0][i]+x

        dM=min(m, n)>>1
        x=[-1]*3
        for d in range(dM+1):
            for i in range(d, m-d):
                for j in range(d, n-d):
                    y=rhombusSum(i, j, d)
                    if y==x[0] or y==x[1] or y==x[2]: continue
                    if y>x[0]:
                        x[2]=x[1]
                        x[1]=x[0]
                        x[0]=y
                    elif y>x[0]:
                        x[2]=x[1]
                        x[1]=x[0]
                        x[0]=y
                    elif y>x[1]:
                        x[2]=x[1]
                        x[1]=y
                    elif y>x[2]:
                        x[2]=y

        for i in range(2, -1, -1):
            if x[i]==-1: x.pop()

        return x
```

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(m * n * min(m,n))
```

- For each radius `d`, we iterate over possible centers.
- Border sum computation is **O(1)** due to prefix sums.

**Space Complexity**

```
O(m * n)
```

For diagonal prefix arrays.

---

## 🧪 Edge Cases

1. **Single cell grid**
```
grid = [[5]]
```
Output → `[5]`

2. **No rhombus larger than radius 0**
Small grids only produce single-cell rhombuses.

3. **Duplicate sums**
Only **distinct** values are kept.

4. **Large grids**
Prefix sums ensure border sums are computed efficiently.

5. **Negative values in grid**
Still works because prefix sums support negative numbers.
