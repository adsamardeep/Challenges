# 🔢 Check If a String Contains All Binary Codes of Size K

## 📜 Problem Statement

Given a binary string `s` and an integer `k`, return `True` if **every possible binary code of length `k`** appears as a substring of `s`. Otherwise, return `False`.

There are exactly:

```

2^k

```

possible binary codes of length `k`.

---

## 🔍 Example

### Example 1:
**Input:**
```

s = "00110110", k = 2

```

Possible binary codes of length 2:
```

00, 01, 10, 11

```

All appear in `s`.

**Output:**
```

True

```

---

### Example 2:
**Input:**
```

s = "0110", k = 2

```

Missing:
```

00

```

**Output:**
```

False

````

---

## 🧠 Key Idea

Instead of generating all substrings and storing them as strings (slow),  
we use a **rolling hash with bit manipulation**.

---

## 🚀 Step-by-Step Explanation

### 1️⃣ Total Required Codes

```python
req = 1 << k
````

We need `2^k` unique codes.

---

### 2️⃣ Use Bitmask for Sliding Window

```python
mask = req - 1
```

This keeps only the last `k` bits.

---

### 3️⃣ Rolling Hash

```python
h = ((h << 1) & mask) | (ord(ch) & 1)
```

Explanation:

* `h << 1` → shift left (drop oldest bit)
* `& mask` → keep only last `k` bits
* `ord(ch) & 1` → converts `'0'` → 0 and `'1'` → 1

So `h` always stores current `k`-length window as an integer.

---

### 4️⃣ Track Seen Codes

```python
seen = [False] * (1 << k)
```

When a new code appears:

* Mark as seen
* Decrease `req`
* If `req == 0` → all codes found → return True

---

## 💻 Code 

```python
class Solution:
    def hasAllCodes(self, s: str, k: int) -> bool:
        req = 1 << k
        seen = [False] * req
        mask = req - 1
        h = 0

        for i, ch in enumerate(s):
            h = ((h << 1) & mask) | (ord(ch) & 1)

            if i >= k - 1 and not seen[h]:
                seen[h] = True
                req -= 1
                if req == 0:
                    return True

        return False
```

---

## ⏱ Complexity Analysis

Let `n = len(s)`

### Time Complexity:

* Single pass through string
* **O(n)**

### Space Complexity:

* Stores up to `2^k` values
* **O(2^k)**

---

## 🧪 Edge Cases

1. `k > len(s)` → False
2. `k = 1` → just check if both '0' and '1' exist
3. `s` shorter than `2^k + k - 1` → automatically False
4. All identical characters

---

## 🎯 Why This Is Optimal

* No substring slicing
* No string hashing
* Pure integer rolling window
* Early stopping when all codes found

This is the optimal interview solution.
