# 🧩 Make Largest Special Binary String

## 📜 Problem Statement

A **special binary string** satisfies:

1. The number of `1`s equals the number of `0`s  
2. For every prefix, `#1 ≥ #0`

Given a special binary string `s`, rearrange its special substrings to make it **lexicographically largest**.

---

## 🔍 Example

### Input:
```

s = "11011000"

```

### Output:
```

"11100100"

```

---

## 🧠 Intuition

A special string behaves exactly like a **valid parentheses string**:

- `1` → "("  
- `0` → ")"  

So the problem becomes:
> Rearrange valid parentheses substrings to get the lexicographically largest result.

---

## 🚀 Approach (Step-by-Step)

### 1️⃣ Split into Primitive Special Substrings

Use a counter:

- `+1` for `'1'`
- `-1` for `'0'`

When `count == 0`, we found a **primitive special block**.

Example:
```

"11011000"

```

Breakdown:
```

1 ( 1 0 1 1 0 0 )
↑             ↑
Balanced block found

```

---

### 2️⃣ Recursively Optimize Inner Part

For each balanced block:
```

1 + makeLargestSpecial(inner) + 0

```

This ensures:
- The inside is already lexicographically largest
- The whole block remains special

---

### 3️⃣ Sort Blocks in Reverse Order

To maximize lexicographical order:

```

res.sort(reverse=True)

```

Why?
- Larger blocks should come first
- Greedy choice works because each block is valid and independent

---

### 4️⃣ Join Everything

Finally:
```

return ''.join(res)

````

---

## 💻 Code 

```python
class Solution:
    def makeLargestSpecial(self, s: str) -> str:
        count = 0
        i = 0
        res = []

        for j in range(len(s)):
            if s[j] == '1':
                count += 1
            else:
                count -= 1

            if count == 0:
                inner = self.makeLargestSpecial(s[i+1:j])
                res.append('1' + inner + '0')
                i = j + 1

        res.sort(reverse=True)

        return ''.join(res)
````

---

## ⏱ Complexity Analysis

Let `n = len(s)`

### Time Complexity:

* Each level splits string and sorts partitions
* Worst case ≈ **O(n²)**

### Space Complexity:

* Recursion depth up to `n`
* **O(n)**

---

## 🧪 Edge Cases

1. `"10"` → `"10"`
2. `"1100"` → `"1100"`
3. Already largest → unchanged
4. Deep nesting → recursive optimization
5. Single primitive block → no sorting effect

---

## 🎯 Why This Works

* Special strings are structurally recursive
* Each primitive block is independent
* Sorting ensures greedy lexicographic maximization
* Recursion ensures inner optimality first
