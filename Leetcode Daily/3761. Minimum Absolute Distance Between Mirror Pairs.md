# 🏷 Minimum Mirror Pair Distance

## 📜 Problem Statement  
This function finds the minimum distance between two indices such that one number is the reverse (mirror) of the other. While traversing the array, it checks if the current number has previously appeared as a reversed value. If no such pair exists, the function returns `-1`.

## 🔍 Example Input/Output  
Input:  
nums = [12, 34, 21, 43]  

Output:  
2  

Explanation:  
- Reverse of `12` is `21`, found at index `2` → distance = `2`  
- Reverse of `34` is `43`, found at index `3` → distance = `2`  
Minimum distance = `2`

## 🧠 Approach  
1. Initialize a dictionary `seen` to store reversed values mapped to their indices.  
2. Traverse the array:
   - For each number `n`, check if it already exists in `seen`.  
   - If yes, compute the distance between current index and stored index.  
3. Update the result with the minimum distance found.  
4. Store the reversed version of the current number (`str(n)[::-1]`) in `seen` with its index.  
5. After traversal:
   - If no valid pair found, return `-1`.  
   - Otherwise, return the minimum distance.

## 💻 Code 
```python
class Solution:
    def minMirrorPairDistance(self, nums: List[int]) -> int:
        res = 100000
        seen = {}

        for i, n in enumerate(nums):
            if n in seen:
                res = min(res, i - seen[n])
            seen[int(str(n)[::-1])] = i

        return -(res == 100000) | res
````

## ⏱ Complexity Analysis

Time Complexity: O(n * d), where `d` is the number of digits (for reversing numbers).
Space Complexity: O(n), for storing seen reversed values.

## 🧪 Edge Cases

* No mirror pairs → returns `-1`.
* Number is a palindrome (e.g., 121) → matches itself if seen before.
* Leading zeros after reversal (e.g., 10 → 1) → handled correctly by integer conversion.
* Multiple mirror pairs → returns the minimum distance.
* Single element → returns `-1`.
