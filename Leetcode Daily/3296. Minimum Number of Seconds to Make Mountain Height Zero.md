## 🏷 Minimum Seconds to Build a Mountain Using Binary Search

## 📜 Problem Statement
You are given an integer `height` representing the total height of a mountain that needs to be built, and an array `times` where `times[i]` is the time coefficient of the `i-th` worker.  
A worker who takes `t` seconds per unit builds heights in increasing time: `t, 2t, 3t, ...`. Determine the **minimum number of seconds** required for all workers combined to build a mountain of height `height`.

## 🔍 Example Input/Output

**Example**

Input:
```

height = 5
times = [1, 2]

```

Output:
```

6

```

Explanation:

Worker 1 (`t=1`) builds heights taking time:
```

1 + 2 + 3 = 6 seconds → height 3

```

Worker 2 (`t=2`) builds heights taking time:
```

2 + 4 = 6 seconds → height 2

```

Total height = `3 + 2 = 5`, which meets the requirement.

---

## 🧠 Approach

### 1. Observation
For a worker with time `t`, building `k` levels requires:

```

t * (1 + 2 + ... + k) = t * k * (k + 1) / 2

```

We need the largest `k` such that:

```

t * k * (k + 1) / 2 ≤ mid

```

### 2. Solve for `k`
Rearranging the inequality:

```

k(k+1)/2 ≤ mid / t

```

This quadratic equation gives:

```

k ≈ floor( sqrt(2 * mid / t + 0.25) - 0.5 )

````

### 3. Binary Search
- Search the minimum time `T` such that the workers can collectively build at least `height`.
- For each candidate time `mid`:
  - Compute how many levels each worker can build.
  - Sum them.
- If total ≥ `height`, reduce search space.
- Otherwise increase time.

### 4. Result
The smallest `mid` satisfying the condition is the answer.

---

## 💻 Code 

```python
class Solution:
    def minNumberOfSeconds(self, height: int, times: list[int]) -> int:
        lo, hi = 1, 10**16

        while lo < hi:
            mid = (lo + hi) >> 1
            tot = 0
            for t in times:
                tot += int(math.sqrt(mid / t * 2 + 0.25) - 0.5)
                if tot >= height: break
            if tot >= height:
                hi = mid
            else:
                lo = mid + 1

        return lo
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(W log T)
```

Where:

* `W` = number of workers (`len(times)`)
* `T` = search range (`10^16`)

Binary search runs about **~54 iterations**, checking all workers each time.

**Space Complexity**

```
O(1)
```

Only constant extra variables are used.

---

## 🧪 Edge Cases

1. **Single worker**

```
height = 3
times = [2]
```

2. **Large height**
   Algorithm still works because of binary search.

3. **Workers with different speeds**
   Faster workers contribute more height.

4. **Very slow workers**
   They may contribute zero height for small `mid`.

5. **Minimum height**

```
height = 1
```

Only the fastest worker determines the answer.
