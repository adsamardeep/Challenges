# 🧮 Minimum Swaps to Arrange a Binary Grid

## 📜 Problem Statement

You are given an `n x n` binary grid.

In one move, you can swap **adjacent rows**.

Return the minimum number of swaps required so that for every row `i`:

```

Number of trailing zeros ≥ (n - 1 - i)

```

If it’s impossible, return `-1`.

---

## 🔍 Why Trailing Zeros Matter

For row `i` (0-indexed from top),  
it must have at least:

```

n - 1 - i

````

trailing zeros.

Example for `n = 3`:

| Row index | Required trailing zeros |
|-----------|------------------------|
| 0         | 2                      |
| 1         | 1                      |
| 2         | 0                      |

---

## 🧠 Step-by-Step Explanation

### 1️⃣ Compute Trailing Zeros Per Row

```python
zeros = list(map(lambda r: (r[::-1] + [1]).index(1), grid))
````

Explanation:

* `r[::-1]` → reverse row
* Append `[1]` to avoid errors
* `.index(1)` → finds first `1`
* That position equals number of trailing zeros

Example:

```
[1,0,0] → reversed → [0,0,1]
index(1) = 2 → trailing zeros = 2
```

---

### 2️⃣ Greedy Row Placement

For each row `i`:

We need a row whose trailing zeros:

```
≥ n - 1 - i
```

We:

* Find the first row satisfying this
* Count how many swaps needed to bring it to top
* Remove that row from list
* Continue

---

### 3️⃣ If No Valid Row Found

Return:

```
-1
```

---

## 💻 Code 

```python
from typing import List

class Solution:
    def minSwaps(self, grid: List[List[int]]) -> int:
        n = len(grid)
        zeros = list(map(lambda r: (r[::-1] + [1]).index(1), grid))
        
        swaps = 0
        for i in range(n):
            j = (zeros + [n]).index(
                next(filter(lambda v: v >= n - 1 - i, zeros + [n]))
            )
            if j == len(zeros):
                return -1
            swaps += j
            zeros.pop(j)
        
        return swaps
```

---

## ⏱ Complexity Analysis

Let `n` be grid size.

### Time Complexity:

* Trailing zero calculation → O(n²)
* Nested scanning → O(n²)
* Overall → **O(n²)**

### Space Complexity:

* Store `zeros` list → O(n)

---

## 🧪 Edge Cases

1. Already valid grid → 0 swaps
2. Impossible configuration → -1
3. Single row → 0
4. All zeros grid
5. All ones grid

---

## 🎯 Why This Greedy Works

* Always place the earliest valid row
* Minimize swaps by picking closest candidate
* Equivalent to bubble-swapping into position
