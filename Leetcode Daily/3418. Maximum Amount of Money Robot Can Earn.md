## 🏷 Maximum Coins with Up to Two Skips (Grid DP Optimization)

## 📜 Problem Statement
Given a 2D grid `coins`, you traverse row by row (left to right within each row). At each cell, you can either **collect the coin value** or **skip collecting it**.  
You are allowed to skip collecting coins **at most 2 times total**. Return the **maximum sum of collected coins**.

## 🔍 Example Input/Output

**Example**

Input:
```

coins = [
[1, -2, 3],
[4, 5, -6]
]

```

Output:
```

13

```

Explanation:
One optimal strategy:
- Take: 1, skip -2, take 3
- Take: 4, take 5, skip -6  
Total = 1 + 3 + 4 + 5 = 13

---

## 🧠 Approach

### 1. DP State Definition
We maintain:
```

dp[j][k]

```
Where:
- `j` → column index (1-based)
- `k` → number of skips used (0, 1, or 2)
- Value = maximum sum achievable up to this point

---

### 2. Initialization
```

dp[1] = [0, 0, 0]

```

- Start with zero sum before processing grid.

---

### 3. Transition Logic
For each cell value `x`, update DP from left (`j`) and top (`j+1`):

#### Case 1: No skip used (k = 0)
```

dp[j+1][0] = max(dp[j][0], dp[j+1][0]) + x

```

---

#### Case 2: One skip used (k = 1)
Either:
- Continue taking:
```

dp[j][1] + x or dp[j+1][1] + x

```
- Or skip current cell:
```

dp[j][0] or dp[j+1][0]

```

---

#### Case 3: Two skips used (k = 2)
Either:
- Continue taking:
```

dp[j][2] + x or dp[j+1][2] + x

```
- Or skip:
```

dp[j][1] or dp[j+1][1]

```

---

### 4. Iteration Strategy
- Process row by row
- Update DP left-to-right

---

### 5. Final Answer
```

dp[n][2]

````

- Maximum sum using up to 2 skips

---

## 💻 Code 

```python
class Solution:
    def maximumAmount(self, coins: List[List[int]]) -> int:
        n = len(coins[0])
        dp = [[-inf] * 3 for _ in range(n + 1)]
        dp[1] = [0] * 3
        for row in coins:
            for j, x in enumerate(row):
                dp[j + 1][2] = max(dp[j][2] + x, dp[j + 1][2] + x, dp[j][1], dp[j + 1][1])
                dp[j + 1][1] = max(dp[j][1] + x, dp[j + 1][1] + x, dp[j][0], dp[j + 1][0])
                dp[j + 1][0] = max(dp[j][0], dp[j + 1][0]) + x
        return dp[n][2]
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(m * n)
```

* Traverse all cells once

**Space Complexity**

```
O(n)
```

* Only one row of DP stored

---

## 🧪 Edge Cases

1. **All positive values**

```
Best to never skip
```

2. **All negative values**

```
Skip up to 2 worst values
```

3. **Single row**

```
Simple linear DP
```

4. **Single column**

```
Still works with same logic
```

5. **Less than 2 negatives**

```
Unused skips don't affect result
```
