# 🔁 Minimum Flips to Make a Binary String Alternating (With Rotation)

## 📜 Problem Statement

Given a binary string `s`, you can perform two operations:

1. **Rotate the string**  
   Move the first character to the end of the string.

2. **Flip a character**  
   Change `'0' → '1'` or `'1' → '0'`.

Return the **minimum number of flips** required to make the string **alternating**.

An alternating string looks like either:

```

010101...
101010...

```

---

# 🧠 Key Idea

Two observations:

1️⃣ There are **only two valid alternating patterns**:
```

Pattern A: 010101...
Pattern B: 101010...

```

2️⃣ Because we can **rotate**, we must consider **every rotation of `s`**.

Instead of actually rotating the string (which would cost `O(n²)`),  
we simulate it using a **sliding window over positions**.

---

# ⚙️ Trick Used in the Code

The expression:

```

(ord(s[i]) ^ i) & 1

```

checks whether the character matches the expected parity.

Why?

- `ord('0') = 48` (even)
- `ord('1') = 49` (odd)

Thus:
```

parity of character = ord(s[i]) & 1

```

If `(ord(s[i]) ^ i) & 1` is:

- `0` → correct for pattern `"010101..."`
- `1` → mismatch

So we count mismatches efficiently using bit operations.

---

# 🚀 Step-by-Step Algorithm

### 1️⃣ Count mismatches for initial string

```

op[0] = mismatches for pattern "0101..."
op[1] = mismatches for pattern "1010..."

```

---

### 2️⃣ Slide rotation window

For each rotation:

- Remove contribution of `s[i]` from front
- Add contribution when it moves to the end
- Update mismatch counts

```

op[(c ^ i) & 1] -= 1
op[(c ^ (n + i)) & 1] += 1

```

---

### 3️⃣ Track minimum flips

```

res = min(res, min(op))

````

Because we choose the better of the two patterns.

---

# 💻 Code

```python
class Solution:
    def minFlips(self, s: str) -> int:
        n = len(s)
        res = n
        op = [0, 0]

        for i in range(n):
            op[(ord(s[i]) ^ i) & 1] += 1

        for i in range(n):
            c = ord(s[i])
            op[(c ^ i) & 1] -= 1
            op[(c ^ (n + i)) & 1] += 1
            res = min(res, min(op))

        return res
````

---

# ⏱ Complexity Analysis

### Time Complexity

```
O(n)
```

* One pass for mismatch count
* One pass for rotations

---

### Space Complexity

```
O(1)
```

Only a few counters are used.

---

# 🔎 Example

Input:

```
s = "111000"
```

Possible rotations:

```
111000
110001
100011
000111
001110
011100
```

Best alternating form requires **2 flips**.

Output:

```
2
```

---

# 🎯 Why This Solution Is Smart

* Avoids generating rotations (`O(n²)`)
* Uses **bit tricks for mismatch checking**
* Maintains mismatch counts in **constant time per rotation**
