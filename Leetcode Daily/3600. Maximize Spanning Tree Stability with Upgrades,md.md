## 🏷 Maximum Stability of a Network with Limited Edge Upgrades

## 📜 Problem Statement
You are given a graph with `n` nodes and a list of edges. Each edge is represented as `[u, v, w, must]`, where `u` and `v` are endpoints, `w` is the stability weight, and `must` indicates whether the edge **must be included** (`1`) or is optional (`0`). You may upgrade at most `k` optional edges, doubling their weight. The goal is to build a **spanning network** while maximizing the **minimum stability value** among all chosen edges.

## 🔍 Example Input/Output

**Example**

Input:
```

n = 3
edges = [
[0,1,4,1],
[1,2,3,0],
[0,2,2,0]
]
k = 1

```

Output:
```

4

````

Explanation:
- The required edge `(0,1)` with weight `4` must be included.
- We connect the remaining node using `(1,2)` with weight `3`.
- Since we can upgrade one optional edge, weight `3 → 6`.
- The minimum stability among selected edges becomes `min(4,6) = 4`.

---

## 🧠 Approach

This algorithm combines **Union-Find (Disjoint Set Union)**, **radix sorting**, and a **greedy edge selection strategy**.

### 1. Union-Find Initialization
- Each node starts as its own parent.
- The `find` function uses **path compression** for efficiency.

### 2. Process Mandatory Edges
- All edges with `must == 1` must be included.
- If two nodes are already connected, the graph forms a cycle → return `-1`.
- Track the smallest weight among these edges (`min_m`).

### 3. Collect Optional Edges
- For edges with `must == 0`, only consider those connecting different components.
- Pack `(weight, root_u, root_v)` into a single integer to allow fast sorting.

### 4. Radix Sort
- Perform radix sorting on packed edges by weight using byte-level counting sort.
- This avoids the overhead of comparison-based sorting.

### 5. Build the Remaining MST
- Process edges in descending weight order.
- If an edge connects two different components, merge them.
- Store selected edge weights for potential upgrades.

### 6. Apply Upgrades
- Up to `k` edges can be upgraded (weight doubled).
- Update the minimum stability accordingly.

### 7. Final Result
- If all nodes become connected, return the **minimum stability value** among selected edges.
- Otherwise, return `-1`.

## 💻 Code 

```python
class Solution:
    def maxStability(self, n: int, edges: list[list[int]], k: int) -> int:
        parent = list(range(n))

        def find(i):
            while i != parent[i]:
                parent[i] = parent[parent[i]]
                i = parent[i]
            return i

        min_m = float('inf')
        comp_count = n

        for u, v, w, must in edges:
            if must == 1:
                ru = find(u)
                rv = find(v)
                if ru == rv:
                    return -1
                parent[ru] = rv
                comp_count -= 1
                if w < min_m:
                    min_m = w

        opt = []
        for u, v, w, must in edges:
            if must == 0:
                ru = find(u)
                rv = find(v)
                if ru != rv:
                    opt.append((w << 34) | (ru << 17) | rv)

        if comp_count == 1:
            return int(min_m)

        if len(opt) > 1:
            temp = [0] * len(opt)
            for shift in range(34, 64, 8):
                count = [0] * 256
                for val in opt:
                    count[(val >> shift) & 0xFF] += 1
                for i in range(1, 256):
                    count[i] += count[i - 1]
                for i in range(len(opt) - 1, -1, -1):
                    val = opt[i]
                    idx = (val >> shift) & 0xFF
                    count[idx] -= 1
                    temp[count[idx]] = val
                opt, temp = temp, opt

        upg = []
        for i in range(len(opt) - 1, -1, -1):
            packed = opt[i]
            ru = find((packed >> 17) & 0x1FFFF)
            rv = find(packed & 0x1FFFF)
            
            if ru != rv:
                parent[ru] = rv
                upg.append(packed >> 34)
                comp_count -= 1
                if comp_count == 1:
                    break

        if comp_count > 1:
            return -1

        for i in range(len(upg) - 1, -1, -1):
            if k > 0:
                k -= 1
                upgraded = upg[i] << 1
                if upgraded < min_m:
                    min_m = upgraded
            else:
                if upg[i] < min_m:
                    min_m = upg[i]
                break

        return int(min_m)
````

## ⏱ Complexity Analysis

**Time Complexity**

```
O(E + N + E log W)
```

* Union-Find operations: near `O(E α(N))`
* Radix sort processes edges in linear passes over bytes.

**Space Complexity**

```
O(N + E)
```

* Parent array for Union-Find.
* Temporary arrays for radix sorting.

## 🧪 Edge Cases

1. **Cycle in mandatory edges**

```
If must-edges already form a cycle → return -1
```

2. **Graph already connected with mandatory edges**

```
Return the minimum mandatory edge weight
```

3. **Not enough edges to connect all nodes**

```
Return -1
```

4. **No upgrades allowed**

```
k = 0 → use original edge weights
```

5. **All edges optional**

```
Algorithm behaves like building a maximum spanning tree with upgrades
```
