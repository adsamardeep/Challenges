# 🔢 Number of Steps to Reduce a Binary Number to One

## 📜 Problem Statement

Given a binary string `s` representing a positive integer:

Repeat until the number becomes `"1"`:

- If even → divide by 2  
- If odd → add 1  

Return the number of steps required.

---

## 🔍 Example

### Example 1:
**Input:**
```

s = "1101"

```

**Process:**
```

1101 (13) → odd → +1 → 1110 (14)
1110 → even → /2 → 111 (7)
111 → odd → +1 → 1000 (8)
1000 → even → /2 → 100
100 → even → /2 → 10
10 → even → /2 → 1

````

**Steps = 6**

---

## 🧠 Key Insight

Instead of converting to an integer (which may overflow for large inputs),  
we simulate the process directly on the binary string using a **carry approach**.

---

## 🚀 Step-by-Step Explanation

We process from **right to left** (LSB to MSB).

We maintain:

- `steps` → total operations
- `carry` → simulates addition carry when we add 1

---

### 🔎 At Each Bit (Right to Left)

For each bit except the first:

```python
((s[i] == '1') + carry) & 1
````

This checks if the current bit (after carry) is **odd**.

---

### Case 1️⃣: Odd (Result is 1)

* We must:

  * Add 1 → causes carry
  * Then divide by 2
* So:

```
steps += 2
carry = 1
```

---

### Case 2️⃣: Even (Result is 0)

* Just divide by 2

```
steps += 1
```

---

### Final Step

After loop:

```python
return steps + carry
```

If there's a leftover carry at the MSB, it adds one more step.

---

## 💻 Code 

```python
class Solution:
    def numSteps(self, s: str) -> int:
        steps = 0
        carry = 0

        for i in range(len(s) - 1, 0, -1):
            if (((s[i] == '1') + carry) & 1):
                steps += 2
                carry = 1
            else:
                steps += 1

        return steps + carry
```

---

## ⏱ Complexity Analysis

Let `n = len(s)`

### Time Complexity:

* Single pass
* **O(n)**

### Space Complexity:

* Constant extra space
* **O(1)**

---

## 🧪 Edge Cases

1. `"1"` → 0
2. `"10"` → 1
3. `"11"` → 3
4. Large binary string (10⁵ length) → works efficiently
5. All 1s like `"11111"` → carry propagates fully

---

## 🎯 Why This Is Optimal

* No integer conversion
* No string rebuilding
* Pure bit logic
* Handles very large inputs efficiently
