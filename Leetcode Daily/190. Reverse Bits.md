# 🔄 Reverse Bits (32-bit Integer)

## 📜 Problem Statement
Given a 32-bit unsigned integer `n`, reverse its bits and return the resulting integer.

The input is treated as a 32-bit value, and the output should also be a 32-bit integer with bits in reverse order.

---

## 🔍 Example Input/Output

### Example 1:
**Input:**  
n = 43261596  

**Binary:**  
00000010100101000001111010011100  

**Output:**  
964176192  

**Binary:**  
00111001011110000010100101000000  

---

### Example 2:
**Input:**  
n = 4294967293  

**Output:**  
3221225471  

---

## 🧠 Approach (Step-by-Step)

This solution uses a **Divide and Conquer (Recursive Bit Reversal)** technique.

### Step 1: Base Case
If the bit length is 1, return the least significant bit:
```python
return v & 1
````

### Step 2: Divide

Split the number into two equal halves:

* `half = length >> 1`
* Create a mask to extract lower half bits:

  ```python
  mask = (1 << half) - 1
  ```
* Extract:

  * `lo` → lower half
  * `hi` → upper half

### Step 3: Conquer

Recursively reverse:

* Reverse lower half
* Reverse upper half

### Step 4: Combine

Swap halves:

```python
(rev(lo, half) << half) | rev(hi, half)
```

This effectively reverses the entire 32-bit number.

---

## 💻 Code 

```python
from typing import List

class Solution:
    def reverseBits(self, n: int) -> int:
        def rev(v: int, length: int) -> int:
            if length == 1:
                return v & 1

            half = length >> 1
            mask = (1 << half) - 1

            lo = v & mask
            hi = v >> half

            return (rev(lo, half) << half) | rev(hi, half)

        return rev(n, 32)
```

---

## ⏱ Complexity Analysis

### Time Complexity:

* Each level splits the problem into 2 halves.
* Total operations = **O(32)**
* Since 32 is constant → **O(1)**

### Space Complexity:

* Recursive stack depth = log₂(32) = 5
* **O(log 32) → O(1)** (constant space)

---

## 🧪 Edge Cases

1. **n = 0** → Output = 0
2. **n = 1** → Output = 2147483648
3. **All bits are 1 (4294967295)** → Output remains same
4. **Leading zeros case** → Properly handled since full 32 bits are processed
5. **Maximum unsigned 32-bit integer**
