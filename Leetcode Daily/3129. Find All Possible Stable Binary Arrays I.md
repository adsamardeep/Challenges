## 🏷 Count Number of Stable Binary Arrays with Limited Consecutive Elements

## 📜 Problem Statement
Given three integers `zero`, `one`, and `limit`, count the number of binary arrays that contain exactly `zero` number of `0`s and `one` number of `1`s. The array is considered **stable** if it does not contain more than `limit` consecutive `0`s or `1`s. Return the total number of such stable arrays modulo `10^9 + 7`.

## 🔍 Example Input/Output

**Example**

Input:
```
zero = 1
one = 1
limit = 1
```

Output:
```
2
```

Explanation:
Possible arrays:
```
01
10
```
Both satisfy the constraint of not having more than `1` consecutive identical element.

---

## 🧠 Approach

This solution combines **combinatorics**, **inclusion–exclusion**, and **precomputed factorials**.

### 1. Precompute Factorials
To efficiently compute combinations (`nCr`) modulo `10^9+7`:
- Precompute factorials `fact`
- Precompute modular inverse factorials `invFact` using Fermat’s Little Theorem.

### 2. Combination Function
Define:

```
C(n, k) = n! / (k! * (n-k)!)
```

Computed using the precomputed factorial and inverse factorial arrays.

### 3. Counting Valid Segment Arrangements
Define function:

```
F(N, K, L)
```

Where:
- `N` = number of elements
- `K` = number of segments
- `L` = maximum allowed consecutive elements

We distribute `N` elements into `K` segments such that each segment length ≤ `L`.

This is calculated using **inclusion–exclusion**:
- First count unrestricted distributions.
- Then subtract configurations where segments exceed `L`.

### 4. Combine Zero and One Arrangements
Let:
- `k` = number of zero segments.

Possible one-segment counts around zero segments are:
```
k-1, k, k+1
```

So the number of valid combinations for ones is:

```
fOne[k-1] + 2*fOne[k] + fOne[k+1]
```

### 5. Final Answer
For every possible `k`:

```
answer += F(zero, k, limit) * validOneConfigurations
```

Take modulo `10^9 + 7`.

---

## 💻 Code 

```python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 1000000007
        maxN = zero + one
        
        fact = [0] * (maxN + 1)
        invFact = [0] * (maxN + 1)
        
        fact[0] = 1
        invFact[0] = 1
        for i in range(1, maxN + 1):
            fact[i] = (fact[i - 1] * i) % MOD
            
        invFact[maxN] = pow(fact[maxN], MOD - 2, MOD)
        for i in range(maxN - 1, 0, -1):
            invFact[i] = (invFact[i + 1] * (i + 1)) % MOD
            
        def C(n, k):
            if k < 0 or k > n:
                return 0
            return fact[n] * invFact[k] % MOD * invFact[n - k] % MOD

        def F(N, K, L):
            if K <= 0 or K > N:
                return 0
            ans = 0
            maxJ = (N - K) // L
            for j in range(maxJ + 1):
                term = C(K, j) * C(N - j * L - 1, K - 1) % MOD
                if j & 1:
                    ans = (ans - term + MOD) % MOD
                else:
                    ans = (ans + term) % MOD
            return ans

        maxK = min(zero, one + 1)
        fOne = [0] * (maxK + 2)
        for k in range(1, maxK + 2):
            fOne[k] = F(one, k, limit)
            
        ans = 0
        for k in range(1, maxK + 1):
            fz = F(zero, k, limit)
            if fz == 0:
                continue
            fo = (fOne[k - 1] + 2 * fOne[k] + fOne[k + 1]) % MOD
            ans = (ans + fz * fo) % MOD
            
        return ans
```

---

## ⏱ Complexity Analysis

**Time Complexity:**  
```
O((zero + one) + min(zero, one) * (N / limit))
```
- Factorial preprocessing: `O(zero + one)`
- Inclusion–exclusion loops for segment counting.

**Space Complexity:**  
```
O(zero + one)
```
For factorial and inverse factorial arrays.

---

## 🧪 Edge Cases

1. **Only zeros**
```
zero = 5, one = 0
```
Valid only if `limit ≥ 5`.

2. **Only ones**
```
zero = 0, one = 4
```
Valid only if `limit ≥ 4`.

3. **limit = 1**
```
No two adjacent elements can be the same.
```
The array must strictly alternate.

4. **limit ≥ max(zero, one)**
```
Constraint becomes irrelevant.
```
The answer becomes simple combinations.

5. **Large inputs**
The algorithm uses modular arithmetic and precomputation to remain efficient.
