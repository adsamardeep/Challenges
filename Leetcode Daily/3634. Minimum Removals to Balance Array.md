You are given an integer array `nums` and an integer `k`.

You may remove any number of elements from the array.  
The goal is to make the remaining array **valid**, meaning:

> For the minimum element `min` and maximum element `max` in the remaining array:  
> `min * k >= max`

Return the **minimum number of removals** required to achieve this condition.

**Explanation**

* After removing `1` and `2`, remaining array = `[3, 5, 6]`

## ğŸ§  Approach

This is a classic **two-pointer + sorting** problem.

### Key Observations

* Sorting helps compare smallest and largest efficiently.
* For a fixed right pointer `r`, we want the **smallest valid left pointer `l`** such that:

  ```
  nums[l] * k >= nums[r]
  ```
* Any elements **outside** `[l â€¦ r]` must be removed.

### Steps

1. Sort the array
2. If the entire array already satisfies the condition â†’ return `0`
3. Use two pointers:

   * `l` moves forward when condition is violated
   * `r` expands the window
4. Compute removals as:

   ```
   removals = elements_before_l + elements_after_r
            = l + (n - r - 1)
   ```
5. Minimize removals across all valid windows

---

## ğŸ’» Code 

```python
class Solution:
    def minRemoval(self, nums: List[int], k: int) -> int:
        nums.sort()
        if nums[0] * k >= nums[-1]:
            return 0

        n = len(nums)
        ans = n
        l = 0

        for r, x in enumerate(nums):
            while l < r and nums[l] * k < x:
                l += 1
            ans = min(ans, n - r + l - 1)

        return ans
```
