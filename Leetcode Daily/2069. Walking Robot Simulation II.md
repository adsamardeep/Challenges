# 🏷 Walking Robot Simulation II

## 📜 Problem Statement  
This class simulates a robot moving along the perimeter of a rectangular grid of given width and height. The robot starts at position (0,0) facing East and moves along the boundary in a clockwise manner. The task is to support stepping forward a given number of steps and querying the robot’s current position and direction.

## 🔍 Example Input/Output  
Input:  
Robot(4, 3)  
step(2) → getPos() → [2, 0], getDir() → "East"  

Output:  
Position: [2, 0]  
Direction: "East"  

## 🧠 Approach  
1. Initialize the robot at (0,0) facing East.  
2. Compute the perimeter cycle: `2 * (width + height) - 4`, representing one full loop.  
3. In `step(num)`:
   - Reduce steps using modulo with the perimeter to avoid redundant cycles.  
   - Handle the special case when `num == 0` (full cycle): if at origin, direction becomes South.  
4. Move step-by-step along edges:
   - East → increase x until hitting right boundary.  
   - North → increase y until top boundary.  
   - West → decrease x until left boundary.  
   - South → decrease y until bottom boundary.  
5. At each boundary, update direction clockwise.  
6. Continue until all steps are consumed.  
7. `getPos()` returns current coordinates.  
8. `getDir()` maps direction index to string.

## 💻 Code 
```python
class Robot:

    def __init__(self, width: int, height: int):
        self.w = width
        self.h = height
        self.x = 0
        self.y = 0
        self.dir = 0  # 0=East, 1=North, 2=West, 3=South
        self.per = 2 * (self.w + self.h) - 4

    def step(self, num: int) -> None:
        if self.per == 0:
            return

        num %= self.per

        # special case: full cycle
        if num == 0:
            if self.x == 0 and self.y == 0:
                self.dir = 3  # South
            return

        while num > 0:
            if self.dir == 0:  # East
                move = min(num, self.w - 1 - self.x)
                self.x += move
                num -= move
                if num > 0:
                    self.dir = 1

            elif self.dir == 1:  # North
                move = min(num, self.h - 1 - self.y)
                self.y += move
                num -= move
                if num > 0:
                    self.dir = 2

            elif self.dir == 2:  # West
                move = min(num, self.x)
                self.x -= move
                num -= move
                if num > 0:
                    self.dir = 3

            else:  # South
                move = min(num, self.y)
                self.y -= move
                num -= move
                if num > 0:
                    self.dir = 0

    def getPos(self) -> List[int]:
        return [self.x, self.y]

    def getDir(self) -> str:
        if self.dir == 0:
            return "East"
        elif self.dir == 1:
            return "North"
        elif self.dir == 2:
            return "West"
        else:
            return "South"      


# Your Robot object will be instantiated and called as such:
# obj = Robot(width, height)

# param_3 = obj.getDir()
````

## ⏱ Complexity Analysis

Time Complexity: O(k), where k is the number of direction changes (bounded by perimeter traversal, effectively O(1) per step cycle).
Space Complexity: O(1), constant extra space used.

## 🧪 Edge Cases

* width = 1 or height = 1 → perimeter becomes 0, robot cannot move.
* Large step values → handled efficiently using modulo with perimeter.
* Full cycle (num % perimeter == 0) → direction becomes South if at origin.
* Starting at corners → immediate direction changes handled correctly.
* Repeated calls to step → state persists correctly across calls.
