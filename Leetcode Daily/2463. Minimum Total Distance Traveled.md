# 🏷 Minimum Total Distance Traveled by Robots to Factories

## 📜 Problem Statement  
This function assigns robots to factories such that every robot is repaired by some factory without exceeding any factory’s repair limit. Each robot and factory lies on a number line, and the cost of assigning a robot to a factory is the absolute distance between their positions. The goal is to minimize the total distance traveled by all robots.

## 🔍 Example Input/Output  
Input:  
robot = [0, 4, 6]  
factory = [[2, 2], [6, 2]]  

Output:  
4  

Explanation:  
- Assign robots at positions `0` and `4` to factory at position `2` → cost = `|0-2| + |4-2| = 2 + 2`  
- Assign robot at position `6` to factory at position `6` → cost = `0`  
Total minimum distance = `4`

## 🧠 Approach  
1. Sort both `robot` and `factory` so assignments can be considered in order from left to right.  
2. Use dynamic programming with memoization where `solve(i, j)` represents the minimum cost to repair robots starting from index `i` using factories starting from index `j`.  
3. Base cases:
   - If all robots are assigned (`i == n`), the cost is `0`.
   - If all factories are exhausted but robots remain (`j == m`), return infinity since this assignment is impossible.  
4. For each state `(i, j)`:
   - Option 1: Skip the current factory and try assigning robots using later factories.
   - Option 2: Assign `1` to `limit` robots to the current factory, as long as robots remain.
5. While assigning robots to the current factory, keep a running sum of distances instead of recomputing from scratch.  
6. Recursively solve the remaining subproblem for the next robots and next factory.  
7. Store results in `dp[i][j]` so repeated states are computed only once.  
8. Return `solve(0, 0)` as the minimum total distance.

## 💻 Code 
```python
class Solution:
    def minimumTotalDistance(self, robot: List[int], factory: List[List[int]]) -> int:
        robot.sort()
        factory.sort()

        n = len(robot)
        m = len(factory)
        INF = float('inf')

        dp = [[-1] * (m + 1) for _ in range(n + 1)]

        def solve(i: int, j: int) -> int:
            if i == n:
                return 0

            if j == m:
                return INF

            if dp[i][j] != -1:
                return dp[i][j]

            ans = solve(i, j + 1)

            distance = 0
            pos, limit = factory[j]

            for k in range(limit):
                if i + k >= n:
                    break

                distance += abs(robot[i + k] - pos)

                next_cost = solve(i + k + 1, j + 1)

                if next_cost != INF:
                    ans = min(ans, distance + next_cost)

            dp[i][j] = ans
            return ans

        return solve(0, 0)
````

## ⏱ Complexity Analysis

Time Complexity: O(n * m * L), where `n` is the number of robots, `m` is the number of factories, and `L` is the average factory limit considered in transitions. In the worst case, each DP state tries multiple robot assignments for one factory.

Space Complexity: O(n * m), for the memoization table, plus recursion stack depth up to O(n + m).

## 🧪 Edge Cases

* No robots → answer is `0` since nothing needs repair.
* No factories but robots exist → impossible state handled with `INF`.
* A factory with limit `0` → effectively skipped.
* More factory capacity than needed → algorithm may leave some capacity unused.
* All robots assigned to one factory → works if its limit is large enough.
* Robots and factories already aligned at same positions → total distance can be `0`.
* Unbalanced layouts with distant factories → DP still finds the globally optimal assignment.
