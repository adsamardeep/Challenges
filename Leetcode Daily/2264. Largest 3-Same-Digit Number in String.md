# 🔢 Problem: Largest 3-Digit Good Integer

## 📜 Problem Statement
Given a string `num` representing a large integer,  
find the **largest "good" integer** that appears as a substring.

A **good** integer:
- Has exactly 3 digits.
- All 3 digits are the same.

Return the **largest** such substring found (as a string).  
If no good integer exists, return an empty string.

---

## 🔍 Example 1
**Input**
```python
num = "6777133339"
```

**Explanation**
- Good integers found: `"777"`, `"333"`.
- Largest among them: `"777"`.

**Output**
```
"777"
```

---

## 🔍 Example 2
**Input**
```python
num = "2300019"
```

**Explanation**
- Good integers: `"000"`.
- Only `"000"` exists, so return `"000"`.

**Output**
```
"000"
```

---

## 🧠 Approach
1. Initialize `best` as an empty string.
2. Iterate over all substrings of length 3 in `num`.
3. If all three characters are the same:
   - Compare with `best` and store the lexicographically larger one.
4. Return `best` at the end.

---

## 💻 Code Implementation
```python
class Solution:
    def largestGoodInteger(self, num: str) -> str:
        best = ""
        for i in range(len(num) - 2):
            if num[i] == num[i+1] == num[i+2]:
                best = max(best, num[i:i+3])
        return best
```

---

## ⏱ Complexity Analysis
- **Time Complexity:** O(n) — iterate once through the string.
- **Space Complexity:** O(1) — only store `best`.

---

## 🧪 Edge Cases
- No repeating 3-digit substring → return `""`.
- Multiple identical good integers → return that value.
- Leading zeros like `"000111222"` → `"222"` is largest.
