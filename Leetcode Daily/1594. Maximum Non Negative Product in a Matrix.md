## 🏷 Maximum Product Path in a Grid (Handling Negative Values)

## 📜 Problem Statement
Given a grid of integers (which may include negative values), find the **maximum product** of a path from the top-left corner `(0,0)` to the bottom-right corner `(m-1,n-1)`. You can only move **right or down**.  
Return the maximum product modulo \(10^9 + 7\), or `-1` if the maximum product is negative.

## 🔍 Example Input/Output

**Example**

Input:
```

grid = [
[1,-2,1],
[1,-2,1],
[3,-4,1]
]

```

Output:
```

8

```

Explanation:
Path:
```

1 → 1 → 3 → -4 → 1 = 12

```
But optimal path considering sign flips gives maximum positive product = `8`.

---

## 🧠 Approach

### 1. Key Challenge
- Negative numbers can flip the sign of the product.
- A **minimum product can become maximum** when multiplied by a negative number.

### 2. DP State Definition
Maintain two DP tables:

```

maxDp[i][j] → maximum product reaching (i, j)
minDp[i][j] → minimum product reaching (i, j)

```

### 3. Initialization
- Start from `(0,0)`:
```

maxDp[0][0] = minDp[0][0] = grid[0][0]

```

- First row and first column:
  Only one path exists, so propagate multiplication.

### 4. Transition
For each cell `(i, j)`:

```

candidates = [
maxDp[i-1][j] * val,
minDp[i-1][j] * val,
maxDp[i][j-1] * val,
minDp[i][j-1] * val
]


Then:
```

maxDp[i][j] = max(candidates)
minDp[i][j] = min(candidates)



### 5. Final Result
- If final maximum product is negative → return `-1`
- Else return:
```

result % (10^9 + 7)


---

## 💻 Code 

```python
class Solution:
    def maxProductPath(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        MOD = 10**9 + 7

        maxDp = [[0]*n for _ in range(m)]
        minDp = [[0]*n for _ in range(m)]

        maxDp[0][0] = minDp[0][0] = grid[0][0]

        for i in range(1, m):
            maxDp[i][0] = minDp[i][0] = maxDp[i-1][0] * grid[i][0]

        for j in range(1, n):
            maxDp[0][j] = minDp[0][j] = maxDp[0][j-1] * grid[0][j]

        for i in range(1, m):
            for j in range(1, n):
                val = grid[i][j]

                candidates = [
                    maxDp[i-1][j] * val,
                    minDp[i-1][j] * val,
                    maxDp[i][j-1] * val,
                    minDp[i][j-1] * val
                ]

                maxDp[i][j] = max(candidates)
                minDp[i][j] = min(candidates)

        res = maxDp[m-1][n-1]
        return -1 if res < 0 else res % MOD
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(m * n)
```

* Each cell is processed once.

**Space Complexity**

```
O(m * n)
```

* Two DP matrices are maintained.

---

## 🧪 Edge Cases

1. **Single cell**

```
grid = [[-5]]
```

Output → `-1`

2. **Contains zero**

```
grid = [[1,0],[2,3]]
```

Zero resets product path.

3. **All negative numbers**
   Even count → positive product possible.

4. **Odd negative count**
   Final product becomes negative → return `-1`.

5. **Large values**
   Handled using modulo at the end.
