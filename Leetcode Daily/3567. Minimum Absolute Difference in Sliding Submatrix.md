## 🏷 Minimum Absolute Difference in Every k×k Submatrix

## 📜 Problem Statement
Given a 2D integer grid and an integer `k`, compute a result matrix where each cell represents the **minimum absolute difference between distinct elements** in the corresponding `k × k` submatrix.  
If all elements in a submatrix are identical (or only one unique value exists), return `0` for that position.

## 🔍 Example Input/Output

**Example**

Input:
```

grid = [
[1,3,6],
[4,2,8],
[7,5,9]
]
k = 2

```

Output:
```

[
[1,1],
[1,1]
]

```

Explanation:
Each 2×2 submatrix is evaluated:
- Example: `[1,3,4,2] → unique sorted = [1,2,3,4] → min diff = 1`

---

## 🧠 Approach

### 1. Sliding Window over Columns
- For each starting row `i`, process all `k × k` windows horizontally.
- Maintain a list `elements` representing the current window.

### 2. First Window Initialization
- For `j = 0`, build the full `k × k` window from scratch.

### 3. Slide Window Right
For each next column:
- **Remove** elements from the left column (`j-1`)
- **Add** elements from the new right column (`j+k-1`)

This avoids recomputing the entire window.

### 4. Compute Minimum Absolute Difference
- Convert elements to a set → remove duplicates.
- Sort the unique elements.
- Compute:
```

min(temp[i+1] - temp[i])

````

### 5. Store Result
- If only one unique element → result = 0
- Otherwise, store computed minimum difference.

---

## 💻 Code 

```python
class Solution:
    def minAbsDiff(self, grid: list[list[int]], k: int) -> list[list[int]]:
        n, m = len(grid), len(grid[0])
        res = [[0] * (m - k + 1) for _ in range(n - k + 1)]

        for i in range(n - k + 1):
            elements = []

            for j in range(m - k + 1):
                if j == 0:
                    # First window: build from scratch
                    elements = [grid[r][c] for r in range(i, i+k) for c in range(k)]
                else:
                    # Remove leaving column (j-1)
                    for r in range(i, i+k):
                        elements.remove(grid[r][j-1])
                    # Add entering column (j+k-1)
                    for r in range(i, i+k):
                        elements.append(grid[r][j+k-1])

                # Sort a COPY — original must stay unsorted for next slide
                temp = sorted(set(elements))

                if len(temp) <= 1:
                    res[i][j] = 0
                else:
                    res[i][j] = min(temp[x+1] - temp[x] for x in range(len(temp)-1))

        return res
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O((n - k + 1) * (m - k + 1) * k^2 log(k^2))
```

* Each window processes up to `k^2` elements.
* Sorting unique elements dominates.

**Space Complexity**

```
O(k^2)
```

* Storage for current window elements.

---

## 🧪 Edge Cases

1. **k = 1**

```
Every submatrix has one element → result = 0
```

2. **All elements same**

```
grid = [[5,5],[5,5]]
```

Output → all zeros

3. **Large k (k = min(n, m))**
   Only one submatrix to evaluate.

4. **Duplicate-heavy window**
   Using `set()` ensures only unique values are considered.

5. **Negative values**
   Works correctly since differences are computed after sorting.
