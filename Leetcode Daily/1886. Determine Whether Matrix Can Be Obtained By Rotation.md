## 🏷 Check Matrix Rotation Equality Using Bitmask Optimization

## 📜 Problem Statement
Given two `n x n` matrices `mat` and `target`, determine if `mat` can be rotated (by 0°, 90°, 180°, or 270°) to become equal to `target`. Return `True` if any rotation matches, otherwise return `False`.

## 🔍 Example Input/Output

**Example**

Input:
```

mat = [
[0,1],
[1,0]
]

target = [
[1,0],
[0,1]
]

```

Output:
```

True

```

Explanation:
Rotating `mat` by 90° clockwise results in `target`.

---

## 🧠 Approach

### 1. Key Idea
Instead of explicitly rotating the matrix multiple times, we **check all 4 rotations simultaneously** using a **bitmask**.

### 2. Bitmask Representation
We use a 4-bit mask:
```

0b1111

```
Each bit represents whether a rotation is still valid:
- Bit 0 → 0° rotation
- Bit 1 → 90° rotation
- Bit 2 → 180° rotation
- Bit 3 → 270° rotation

### 3. Rotation Mapping
For each cell `(i, j)`:

- 0°:
```

target[i][j]

```

- 90°:
```

target[j][n-1-i]

```

- 180°:
```

target[n-1-i][n-1-j]

```

- 270°:
```

target[n-1-j][i]

````

### 4. Validation
- For each cell, compare with all 4 rotated positions.
- If mismatch occurs, **turn off the corresponding bit**.
- If all bits become `0`, return `False` early.

### 5. Final Check
If at least one bit remains set (`m != 0`), a valid rotation exists.

---

## 💻 Code 

```python
class Solution:
    def findRotation(self, mat: List[List[int]], target: List[List[int]]) -> bool:
        n = len(mat)
        m = 0b1111

        for i in range(n):
            for j in range(n):
                if mat[i][j] != target[i][j]: m &= 0b1110
                if mat[i][j] != target[j][n - 1 - i]: m &= 0b1101
                if mat[i][j] != target[n - 1 - i][n - 1 - j]: m &= 0b1011
                if mat[i][j] != target[n - 1 - j][i]: m &= 0b0111
                if m == 0: return False

        return m != 0
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(n^2)
```

* Each cell is checked once against all 4 rotations.

**Space Complexity**

```
O(1)
```

* Only a bitmask variable is used.

---

## 🧪 Edge Cases

1. **Already equal (0° rotation)**

```
mat == target → return True
```

2. **Only 180° matches**
   Handled via rotation mapping.

3. **No valid rotation**
   All bits become `0` → return `False`.

4. **Single element matrix**

```
n = 1
```

Always matches if values are equal.

5. **Large matrix**
   Efficient due to single pass with constant checks.
