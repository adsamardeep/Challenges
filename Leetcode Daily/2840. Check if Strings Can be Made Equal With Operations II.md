## 🏷 Check String Equality with Even-Odd Index Permutations (Bitwise Optimization)

## 📜 Problem Statement
Given two strings `s1` and `s2` of equal length, determine if `s1` can be transformed into `s2` by rearranging characters **only within even indices** and **only within odd indices** separately.  
Return `True` if such a transformation is possible, otherwise `False`.

## 🔍 Example Input/Output

**Example**

Input:
```

s1 = "abcd"
s2 = "cdab"

```

Output:
```

True

```

Explanation:
- Even indices: `"a","c"` → can rearrange to match `"c","a"`
- Odd indices: `"b","d"` → can rearrange to match `"d","b"`

---

## 🧠 Approach

### 1. Key Idea
- Characters at **even indices** can only swap among themselves.
- Characters at **odd indices** can only swap among themselves.

So we need to check:
```

multiset(even indices of s1) == multiset(even indices of s2)
multiset(odd indices of s1) == multiset(odd indices of s2)

```

### 2. Optimized Validation (Bitwise + Sum Trick)
Instead of sorting or counting:

For each index:
- Compute:
```

dx = ord(a) ^ ord(b)      # XOR difference
ds = ord(a)^2 - ord(b)^2  # sum difference

```

Maintain separately:
- Even index accumulators: `eX`, `eS`
- Odd index accumulators: `oX`, `oS`

### 3. Why This Works
- XOR ensures matching characters (detects mismatches)
- Sum of squares ensures correct frequency distribution
- Using both avoids collisions

### 4. Final Check
If all accumulators are zero:
```

eX == oX == eS == oS == 0

````

Then strings are equivalent.

---

## 💻 Code 

```python
class Solution:
    def checkStrings(self, s1: str, s2: str) -> bool:
        eX = oX = eS = oS = 0

        for i, (a, b) in enumerate(zip(s1, s2)):
            v1 = ord(a)
            v2 = ord(b)
            dx = v1 ^ v2
            ds = v1 * v1 - v2 * v2

            if i & 1:
                oX ^= dx
                oS += ds
            else:
                eX ^= dx
                eS += ds

        return not (eX | oX | eS | oS)
````

**Time Complexity**

**Space Complexity**

```
O(1)
```

* Only constant variables used.

## 🧪 Edge Cases

1. **Already equal strings**

```
s1 == s2 → True
```

2. **Different lengths**
   (Not applicable as per constraints)

3. **Only even mismatch**

```
Mismatch in even indices → False
```

4. **Only odd mismatch**

```
Mismatch in odd indices → False
```

5. **Repeated characters**
   Handled correctly due to combined XOR + sum check.
