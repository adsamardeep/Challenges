## 🏷 Count Submatrices with Equal Number of X and Y

## 📜 Problem Statement
Given a 2D grid consisting of characters `'X'`, `'Y'`, and possibly others, count the number of **submatrices starting from the top-left corner (0,0)** such that the number of `'X'` and `'Y'` characters inside the submatrix are **equal** and at least one `'X'` exists.

## 🔍 Example Input/Output

**Example**

Input:
```
grid = [
 ["X","Y"],
 ["Y","X"]
]
```

Output:
```
3
```

Explanation:
Valid submatrices starting at `(0,0)`:
```
["X","Y"] → X=1, Y=1 ✅
["X","Y"]
["Y","X"] → X=2, Y=2 ✅
["X"]
["Y"]
["Y"] → invalid cases skipped
```

---

## 🧠 Approach

### 1. Key Idea
We only consider submatrices that start at `(0,0)`.  
So for each cell `(i, j)`, we track the total number of `'X'` and `'Y'` in the rectangle `(0,0) → (i,j)`.

### 2. Row-wise Prefix Counts
For each row:
- Maintain running counts:
```
rx = number of 'X' in current row up to column j
ry = number of 'Y' in current row up to column j
```

### 3. Column Accumulation
Maintain:
```
sumX[j] = total 'X' count up to row i for column j
sumY[j] = total 'Y' count up to row i for column j
```

So:
```
sumX[j] += rx
sumY[j] += ry
```

This effectively builds a **2D prefix sum**.

### 4. Valid Submatrix Condition
For each `(i, j)`:
- Submatrix `(0,0) → (i,j)` is valid if:
```
sumX[j] == sumY[j] AND sumX[j] > 0
```

### 5. Count Results
- Increment result whenever condition is satisfied.

---

## 💻 Code 

```python
class Solution:
    def numberOfSubmatrices(self, grid: List[List[str]]) -> int:
        rows = len(grid)
        cols = len(grid[0])
        sumX = [0] * cols
        sumY = [0] * cols
        res = 0

        for i in range(rows):
            rx = 0
            ry = 0
            for j in range(cols):
                if grid[i][j] == 'X':
                    rx += 1
                elif grid[i][j] == 'Y':
                    ry += 1
                
                sumX[j] += rx
                sumY[j] += ry
                
                if sumX[j] > 0 and sumX[j] == sumY[j]:
                    res += 1

        return res
```

---

## ⏱ Complexity Analysis

**Time Complexity**
```
O(rows * cols)
```
- Each cell is processed once.

**Space Complexity**
```
O(cols)
```
- Only column-wise prefix arrays are used.

---

## 🧪 Edge Cases

1. **No 'X' present**
```
grid = [["Y","Y"]]
```
Output → `0`

2. **No 'Y' present**
```
grid = [["X","X"]]
```
Output → `0`

3. **Single cell**
```
grid = [["X"]]
```
Output → `0` (needs equal X and Y)

4. **Balanced grid**
```
grid = [["X","Y"]]
```
Output → `1`

5. **Large grid**
Efficient due to linear scan with prefix accumulation.
