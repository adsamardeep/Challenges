## 🏷 Construct Product Matrix Using Prefix and Suffix Products

## 📜 Problem Statement
Given a 2D grid of integers, construct a new matrix where each cell contains the **product of all elements in the grid except the current cell**, modulo `12345`.  
The solution must avoid division and compute results efficiently.

## 🔍 Example Input/Output

**Example**

Input:
```

grid = [
[1,2],
[3,4]
]

```

Output:
```

[
[24,12],
[8,6]
]

```

Explanation:
- For position `(0,0)`: product of `[2,3,4] = 24`
- For `(0,1)`: product of `[1,3,4] = 12`
- And so on.

---

## 🧠 Approach

### 1. Key Idea (No Division)
Instead of computing total product and dividing, use:
- **Prefix product** (product of elements before current)
- **Suffix product** (product of elements after current)

### 2. Flattened Traversal Concept
Treat the grid as a flattened array:
```

Traverse row-wise from start → prefix
Traverse row-wise from end → suffix

```

### 3. Suffix Pass
- Start from bottom-right.
- For each cell:
```

p[i][j] = suffix
suffix *= grid[i][j]


This stores product of all elements **after** current cell.

### 4. Prefix Pass
- Traverse from top-left.
- For each cell:
```

p[i][j] *= prefix
prefix *= grid[i][j]


Now each cell has:
```

prefix * suffix = product of all elements except itself

````

### 5. Modulo Handling
- Apply modulo `12345` at each step to prevent overflow.

---

## 💻 Code 

```python

class Solution:
    def constructProductMatrix(self, grid):
        MOD = 12345
        n, m = len(grid), len(grid[0])
        p = [[0]*m for _ in range(n)]

        suffix = 1
        for i in range(n-1, -1, -1):
            for j in range(m-1, -1, -1):
                p[i][j] = suffix
                suffix = (suffix * (grid[i][j] % MOD)) % MOD

        prefix = 1
        for i in range(n):
            for j in range(m):
                p[i][j] = (p[i][j] * prefix) % MOD
                prefix = (prefix * (grid[i][j] % MOD)) % MOD

        return p
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(n * m)
```

* Two passes over the grid.

**Space Complexity**

```
O(n * m)
```

* Output matrix `p`.

---

## 🧪 Edge Cases

1. **Single element**

```
grid = [[5]]
```

Output → `[[1]]` (product excluding itself)

2. **Contains zero**

* If multiple zeros → all results become `0`
* If one zero → only that position gets non-zero product

3. **Large values**
   Handled via modulo `12345`.

4. **All ones**

```
grid = [[1,1],[1,1]]
```

Output remains all `1`s.

5. **Negative values**
   Handled correctly due to modulo arithmetic.
