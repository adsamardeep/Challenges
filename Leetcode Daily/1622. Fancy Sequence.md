## 🏷 Fancy Sequence with Efficient Range Add and Multiply Operations

## 📜 Problem Statement
Design a data structure that supports a sequence of numbers with four operations: append a value, add a number to all elements, multiply all elements by a number, and retrieve a value at a given index. All operations must be performed efficiently under modulo \(10^9 + 7\). The challenge is to avoid updating every element during `addAll` and `multAll`.

## 🔍 Example Input/Output

Example Operations:
```

Fancy fancy = new Fancy();
fancy.append(2);      // sequence: [2]
fancy.addAll(3);      // sequence: [5]
fancy.append(7);      // sequence: [5,7]
fancy.multAll(2);     // sequence: [10,14]
fancy.getIndex(0);    // return 10
fancy.addAll(3);      // sequence: [13,17]
fancy.append(10);     // sequence: [13,17,10]
fancy.multAll(2);     // sequence: [26,34,20]
fancy.getIndex(0);    // return 26
fancy.getIndex(1);    // return 34
fancy.getIndex(2);    // return 20

```

## 🧠 Approach

### 1. Key Idea
Instead of updating every element during `addAll` and `multAll`, maintain a **global linear transformation**:

```

value = a * stored_value + b

```

Where:
- `a` represents cumulative multiplication.
- `b` represents cumulative addition.

### 2. Append Operation
When inserting a new value `val`, reverse the transformation so it fits the current system:

```

stored_value = (val - b) / a

```

Since division is under modulo, use the **modular inverse of `a`**.

### 3. Add Operation
If we add `inc` to every element:

```

value = a * stored_value + (b + inc)

```

So update:

```

b = b + inc

```

### 4. Multiply Operation
If we multiply everything by `m`:

```

value = (a*m) * stored_value + (b*m)

```

So update:

```

a = a * m
b = b * m

```

### 5. Get Operation
Retrieve the actual value using the transformation:

```

result = a * stored_value + b

````

All operations are computed modulo \(10^9 + 7\).

## 💻 Code 

```python
class Fancy:

    def __init__(self):
        self.mod = 10**9 + 7  
        self.val = []  
        self.a = 1  
        self.b = 0  

    def append(self, val: int) -> None:
        x = (val - self.b + self.mod) % self.mod
        self.val.append(x * pow(self.a, self.mod - 2, self.mod) % self.mod)

    def addAll(self, inc: int) -> None:
        self.b = (self.b + inc) % self.mod

    def multAll(self, m: int) -> None:
        self.a = (self.a * m) % self.mod
        self.b = (self.b * m) % self.mod

    def getIndex(self, idx: int) -> int:
        if idx >= len(self.val):
            return -1  
        return (self.a * self.val[idx] + self.b) % self.mod

# Your Fancy object will be instantiated and called as such:
# obj = Fancy()
# obj.append(val)
# obj.addAll(inc)
# obj.multAll(m)
# param_4 = obj.getIndex(idx)
````

## ⏱ Complexity Analysis

**Time Complexity**

```
append   → O(log MOD)   (due to modular inverse)
addAll   → O(1)
multAll  → O(1)
getIndex → O(1)
```

**Space Complexity**

```
O(n)
```

Where `n` is the number of appended elements.

## 🧪 Edge Cases

1. **Index out of range**

```
getIndex(idx >= len(sequence)) → return -1
```

2. **Multiple multiplications by zero**
   If `multAll(0)` occurs, all values become zero, but the transformation still works correctly.

3. **Large number of operations**
   Efficient because operations avoid iterating over the entire sequence.

4. **Modulo overflow**
   All calculations are performed using modulo (10^9 + 7).

5. **Appending after many transformations**
   The reverse transformation ensures appended values remain correct.
