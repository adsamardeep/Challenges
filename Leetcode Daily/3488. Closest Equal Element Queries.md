# 🏷 Minimum Circular Distance for Query Indices

## 📜 Problem Statement  
This function processes queries on a circular array and, for each queried index, returns the minimum circular distance to another index containing the same value. If the value at that index appears only once in the array, the answer for that query is `-1`.

## 🔍 Example Input/Output  
Input:  
nums = [1, 3, 1, 4, 3]  
queries = [0, 1, 3]  

Output:  
[2, 2, -1]  

Explanation:  
- Index `0` has value `1`, and the nearest other `1` is at index `2` → circular distance = `2`  
- Index `1` has value `3`, and the nearest other `3` is at index `4` → circular distance = `2`  
- Index `3` has value `4`, which appears only once → `-1`

## 🧠 Approach  
1. Let `n` be the length of the array.  
2. Build a dictionary `positions` where each value maps to a sorted list of all indices where it appears.  
3. Create an `answer` array initialized with `-1` for every index.  
4. For each list of positions:
   - If the value appears only once, leave its answer as `-1`.
   - Otherwise, for every occurrence, look at its previous and next occurrence in that list.
5. Since the array is circular, use modulo indexing to wrap around:
   - Previous index: `pos[(i - 1 + m) % m]`
   - Next index: `pos[(i + 1) % m]`
6. Compute the circular distance between two indices as:
   - `abs(a - b)`
   - then take `min(distance, n - distance)`
7. Store the minimum of the distances to the previous and next same-value occurrence for each index.  
8. Finally, return the precomputed answers for the requested query indices.

## 💻 Code 
```python
class Solution:
    def solveQueries(self, nums: List[int], queries: List[int]) -> List[int]:
        n = len(nums)

        positions = {}

        for i, num in enumerate(nums):
            if num not in positions:
                positions[num] = []
            positions[num].append(i)

        answer = [-1] * n

        for pos in positions.values():
            m = len(pos)

            if m == 1:
                continue

            for i in range(m):
                curr = pos[i]

                prev_idx = pos[(i - 1 + m) % m]
                next_idx = pos[(i + 1) % m]

                dist_prev = abs(curr - prev_idx)
                dist_prev = min(dist_prev, n - dist_prev)

                dist_next = abs(curr - next_idx)
                dist_next = min(dist_next, n - dist_next)

                answer[curr] = min(dist_prev, dist_next)

        return [answer[idx] for idx in queries]
````

## ⏱ Complexity Analysis

Time Complexity: O(n + q), where `n` is the length of `nums` and `q` is the number of queries.
Space Complexity: O(n), for storing positions of each value and the precomputed answer array.

## 🧪 Edge Cases

* A value appears only once → answer is `-1` for that index.
* All elements are the same → every index has a nearest equal neighbor at distance `1`.
* Query list is empty → returns an empty list.
* Circular wrap gives shorter path → handled using `min(dist, n - dist)`.
* Multiple repeated values → only adjacent occurrences in the sorted position list need to be checked.
