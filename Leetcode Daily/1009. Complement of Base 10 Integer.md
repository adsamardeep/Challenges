## 🏷 Bitwise Complement of an Integer Using Bit Mask Expansion

## 📜 Problem Statement
Given a non-negative integer `n`, return its **bitwise complement**. The complement flips every bit in the binary representation of `n`, considering only the bits up to the most significant `1`. Leading zeros are ignored in the representation.

## 🔍 Example Input/Output

**Example**

Input:
```

n = 5

```

Output:
```

2

```

Explanation:

```

5  -> 101
~  -> 010

```

Result = `2`

Another Example:

Input:
```

n = 0

```

Output:
```

1

```

Explanation:
Binary representation of `0` is `"0"`, and its complement is `"1"`.

---

## 🧠 Approach

1. **Handle the edge case**  
   - If `n == 0`, return `1` because the complement of `0` (binary `"0"`) is `"1"`.

2. **Create a mask covering all significant bits**  
   - Start with `mask = n`.
   - Expand the mask by repeatedly OR-ing it with right-shifted versions of itself.
   - This fills all bits to the right of the highest set bit with `1`.

3. **Bit Expansion Steps**
```

mask |= mask >> 1
mask |= mask >> 2
mask |= mask >> 4
mask |= mask >> 8
mask |= mask >> 16

```

4. **Compute complement**
- Use:
```

~n & mask

````
- `~n` flips all bits, but also introduces leading ones.
- Masking ensures we keep only the bits up to the highest bit of `n`.

---

## 💻 Code 

```python
class Solution:
 def bitwiseComplement(self, n: int) -> int:
     if n == 0: return 1
     mask = n
     for i in (1, 2, 4, 8, 16):
         mask |= mask >> i
     return ~n & mask
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(1)
```

The loop runs a constant number of iterations.

**Space Complexity**

```
O(1)
```

Only a few variables are used.

---

## 🧪 Edge Cases

1. **Zero input**

```
n = 0
```

Output → `1`

2. **Single-bit number**

```
n = 1  (1 → 0)
```

3. **Power of two**

```
n = 8  (1000 → 0111)
```

4. **All bits set**

```
n = 7  (111 → 000)
```

5. **Large integer**
   The algorithm still runs in constant time due to fixed shift operations.
