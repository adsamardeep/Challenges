# 🔁 Minimum Operations on Binary String

## 📜 Problem Statement (Inferred)

Given:
- A binary string `s`
- An integer `k`

You need to determine the **minimum number of operations** required under parity and grouping constraints derived from `k`.

If it is impossible, return `-1`.

---

## 🧠 Understanding the Code Logic

The solution is driven by:

- Total number of `0`s
- Length of string
- Parity constraints
- Mathematical bounds using ceiling division

---

## 🔎 Step-by-Step Explanation

### 1️⃣ Count Number of Zeros

```python
zero += ~ord(s[i]) & 1
````

This is a clever trick:

* `ord('0') = 48`
* `ord('1') = 49`
* `~ord(s[i]) & 1` evaluates to:

  * `1` for `'0'`
  * `0` for `'1'`

So this counts zeros efficiently using bit operations.

---

### 2️⃣ Trivial Cases

#### If no zeros:

```python
if zero == 0:
    return 0
```

#### If entire string length equals k:

```python
if slen == k:
    return 1 if zero == slen else -1
```

Meaning:

* If all characters are zero → 1 operation
* Otherwise impossible

---

### 3️⃣ Define Base Window

```python
base = slen - k
```

Used for alternative grouping constraints.

---

### 4️⃣ Compute Minimum Odd Operations

```python
odd = max(
    ceil(zero / k),
    ceil((slen - zero) / base)
)
```

Then force it to be **odd**:

```python
odd += ~odd & 1
```

---

### 5️⃣ Compute Minimum Even Operations

```python
even = max(
    ceil(zero / k),
    ceil(zero / base)
)
```

Then force it to be **even**:

```python
even += even & 1
```

---

### 6️⃣ Parity Validations

Two conditions:

```python
if (k & 1) == (zero & 1):
```

```python
if (~zero & 1) == 1:
```

These ensure:

* Operation count parity matches required transformation parity

---

### 7️⃣ Return Result

If no valid case:

```python
return -1
```

Otherwise:

```python
return minimum valid operations
```

---

## 💻 Code 

```python
import math

class Solution:
    def minOperations(self, s: str, k: int) -> int:
        zero = 0
        slen = len(s)

        for i in range(slen):
            zero += ~ord(s[i]) & 1

        if zero == 0:
            return 0

        if slen == k:
            return 1 if zero == slen else -1

        base = slen - k

        odd = max(math.ceil(zero / k), math.ceil((slen - zero) / base))
        odd += ~odd & 1

        even = max(math.ceil(zero / k), math.ceil(zero / base))
        even += even & 1

        res = float('inf')

        if (k & 1) == (zero & 1):
            res = min(res, odd)

        if (~zero & 1) == 1:
            res = min(res, even)

        return -1 if res == float('inf') else res
```

## ⏱ Complexity Analysis

Let `n = len(s)`

### Time Complexity:

* Single pass for zero count → **O(n)**
* Constant math operations → **O(1)**
  **Overall: O(n)**

### Space Complexity:

* Constant variables only
  **O(1)**

---

## 🧪 Edge Cases

1. All ones → 0
2. All zeros and `slen == k` → 1
3. Impossible parity case → -1
4. `k = 1`
5. Large input size

---

## 🎯 Observations

* Uses bit tricks for zero counting
* Uses mathematical lower bounds
* Uses parity alignment to validate feasibility
* Optimized to O(n) time and O(1) space
