# 🏷 Walking Robot Simulation

## 📜 Problem Statement  
The function simulates the movement of a robot on an infinite 2D grid based on a list of commands. The robot can turn left, turn right, or move forward, while avoiding obstacle positions. The goal is to return the maximum squared Euclidean distance from the origin that the robot reaches at any point during the simulation.

## 🔍 Example Input/Output  
Input:  
commands = [4, -1, 3]  
obstacles = []  

Output:  
25  

Explanation:  
The robot moves 4 steps north to (0, 4), turns right, then moves 3 steps east to (3, 4).  
The maximum squared distance from the origin is `3^2 + 4^2 = 25`.

## 🧠 Approach  
1. Convert the list of obstacle coordinates into a set for O(1) lookup.  
2. Store the four possible directions in order so the robot can rotate easily.  
3. Initialize the robot at `(0, 0)` facing north.  
4. Traverse each command:
   - If the command is `-2`, turn left.
   - If the command is `-1`, turn right.
   - Otherwise, move forward one step at a time.
5. While moving forward, check after each step whether the new cell contains an obstacle.
   - If it does, step back and stop processing that movement command.
6. After each command, update the maximum squared distance from the origin using `x*x + y*y`.  
7. Return the maximum squared distance encountered.

## 💻 Code 
```python
class Solution:
    def robotSim(self, commands: List[int], obstacles: List[List[int]]) -> int:
        obSet=set()
        for (x, y) in obstacles:
            obSet.add((x, y))
        dir=[(0, 1), (-1, 0), (0, -1), (1, 0)]
        x, y,dx, dy, face, maxD2=0, 0, 0,1, 0, 0
        for  c in commands:
            if c==-2:
                face=(face+1)&3
                dx=dir[face][0]
                dy=dir[face][1]
            elif c==-1:
                face=(face+3)&3
                dx=dir[face][0]
                dy=dir[face][1]
            else:
                for i in range(c):
                    x+=dx
                    y+=dy
                    if (x, y) in obSet:
                        x-=dx
                        y-=dy
                        break
                maxD2=max(maxD2, x*x+y*y)
        return maxD2
````

## ⏱ Complexity Analysis

Time Complexity: O(n + m), where `m` is the number of obstacles and `n` is the total number of forward steps executed across all commands.
Space Complexity: O(m), for storing the obstacle set.

## 🧪 Edge Cases

* Empty commands → robot never moves, so result is 0.
* Empty obstacles → robot moves freely without blockage.
* Immediate obstacle in front of the robot → movement stops for that command.
* Multiple turns in sequence → only direction changes, no position change.
* Large obstacle list → set lookup keeps obstacle checks efficient.
* Commands with only turns → robot remains at the origin, so result is 0.
