## 🏷 Count Number of Stable Binary Arrays (Dynamic Programming with Run-Length Constraint)

## 📜 Problem Statement
Given three integers `zero`, `one`, and `limit`, count the number of binary arrays containing exactly `zero` zeros and `one` ones. The array is considered **stable** if it does not contain more than `limit` consecutive identical elements (`0`s or `1`s). Return the total number of such arrays modulo \(10^9 + 7\).

## 🔍 Example Input/Output

**Example**

Input:
```

zero = 2
one = 1
limit = 1

```

Output:
```

1

```

Explanation:
Possible permutations of two `0`s and one `1`:
```

001 ❌ (two consecutive zeros > limit)
010 ✅
100 ❌ (two consecutive zeros > limit)

```
Only `"010"` satisfies the constraint.

---

## 🧠 Approach

This solution uses **Dynamic Programming** to count valid arrays while enforcing the consecutive element limit.

### 1. DP State Definition
Let:

```

dp[i][j][0]

```
= number of valid arrays using `i` zeros and `j` ones **ending with 0**

```

dp[i][j][1]

```
= number of valid arrays using `i` zeros and `j` ones **ending with 1**

### 2. Base Cases
- Arrays consisting only of zeros:
```

dp[i][0][0] = 1  for i ≤ limit

```

- Arrays consisting only of ones:
```

dp[0][j][1] = 1  for j ≤ limit

```

### 3. Transition
To append a `0`:
```

dp[i][j][0] =
dp[i-1][j][0] + dp[i-1][j][1] - invalid_cases

```

To append a `1`:
```

dp[i][j][1] =
dp[i][j-1][0] + dp[i][j-1][1] - invalid_cases

```

### 4. Removing Invalid Runs
If adding a `0` would create more than `limit` consecutive zeros, subtract the configurations where that overflow happens.

```

over0 = dp[i-limit-1][j][1]
over1 = dp[i][j-limit-1][0]

```

These represent sequences where a run would exceed the allowed limit.

### 5. Final Result
The valid arrays may end with either `0` or `1`:

```

result = dp[zero][one][0] + dp[zero][one][1]

````

Take modulo \(10^9 + 7\).

---

## 💻 Code 

```python
class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7

        dp = [[[0,0] for _ in range(one+1)] for _ in range(zero+1)]

        for i in range(1, min(zero,limit)+1):
            dp[i][0][0] = 1

        for j in range(1, min(one,limit)+1):
            dp[0][j][1] = 1

        for i in range(1, zero+1):
            for j in range(1, one+1):

                over0 = dp[i-limit-1][j][1] if i-limit >= 1 else 0
                over1 = dp[i][j-limit-1][0] if j-limit >= 1 else 0

                dp[i][j][0] = (
                    dp[i-1][j][0] + dp[i-1][j][1] - over0
                ) % MOD

                dp[i][j][1] = (
                    dp[i][j-1][0] + dp[i][j-1][1] - over1
                ) % MOD

        return (dp[zero][one][0] + dp[zero][one][1]) % MOD
````

---

## ⏱ Complexity Analysis

**Time Complexity**

```
O(zero × one)
```

We fill a DP table for every combination of zeros and ones.

**Space Complexity**

```
O(zero × one)
```

The DP table stores results for all `(zero + 1) × (one + 1)` states.

## 🧪 Edge Cases

1. **Only zeros**

```
zero = 5, one = 0
```

Valid only if `limit ≥ 5`.

2. **Only ones**

```
zero = 0, one = 4
```

Valid only if `limit ≥ 4`.

3. **limit = 1**

```
No two adjacent elements can be the same.
```

The sequence must strictly alternate.

4. **limit ≥ max(zero, one)**

```
The constraint becomes irrelevant.
```

The answer becomes the number of ways to arrange zeros and ones.

5. **Minimal input**

```
zero = 1, one = 1, limit = 1
```

Valid arrays: `"01"`, `"10"`.
