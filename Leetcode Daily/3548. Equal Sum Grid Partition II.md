## 🏷 Partition Grid with Optional Single Cell Adjustment

## 📜 Problem Statement
Given a 2D grid of integers, determine if it can be partitioned into two parts with **equal sums** by making a **single horizontal or vertical cut**.  
Additionally, you are allowed to **remove (discount) at most one cell value** from either side of the partition to make the sums equal, while ensuring the remaining part stays connected.

## 🔍 Example Input/Output

**Example**

Input:
```

grid = [
[2,1],
[1,2]
]

```

Output:
```

True

```

Explanation:
- Total sum = 6
- Direct split may not balance
- Removing a single element from one side can equalize sums.

---

## 🧠 Approach

### 1. Precompute Sums
- `row_sums[i]` → sum of row `i`
- `col_sums[j]` → sum of column `j`
- `total_sum` → total grid sum

### 2. Try Horizontal Cuts
- Iterate over possible row cuts.
- Maintain:
```

top_sum, bottom_sum

```

### 3. Direct Match
If:
```

top_sum == bottom_sum → return True

```

### 4. Handle Imbalance (Discount Logic)

If:
```

top_sum > bottom_sum

```

We try to fix difference:
```

difference = top_sum - bottom_sum

```

Check if we can remove a cell with value = `difference` from:
- Top section (while maintaining connectivity constraints)

### 5. Connectivity Constraints
- If only one row or column, removing middle cells may break connectivity.
- So restrict removable cells:
  - Only edges in narrow grids
  - All cells otherwise

### 6. Reverse Adjustment (Rewind Trick)
- Consider undoing last addition and applying discount on the other side.
- Compute:
```

difference_alt = adjusted difference after rewinding

````

- Check if it exists in the opposite partition.

### 7. Repeat for Vertical Cuts
- Same logic applied using `col_sums`.

### 8. Early Pruning
- If no valid adjustment possible and sums diverge too much, break early.

---

## 💻 Code 

```python
class Solution:
    def canPartitionGrid(self, grid: List[List[int]]) -> bool:
        
        H, W = len(grid), len(grid[0])

        row_sums = list(map(sum, grid))
        col_sums = [ sum(grid[r][c] for r in range(H)) for c in range(W) ]
        total_sum = sum(row_sums)

        # Try cutting rows first
        row_cut = 0
        top_sum = 0
        bottom_sum = total_sum

        for row_cut in range(1, H+1):

            top_sum += row_sums[row_cut-1]
            bottom_sum -= row_sums[row_cut-1]

            if top_sum == bottom_sum:
                return True
            elif top_sum > bottom_sum:

                top_discountable = set()
                if row_cut == 1:
                    top_discountable = { grid[row_cut-1][0], grid[row_cut-1][W-1] }
                elif W == 1:
                    top_discountable = { grid[0][0], grid[row_cut-1][0] }
                else:
                    top_discountable = { grid[r][c] for r in range(row_cut) for c in range(W) }

                difference = top_sum - bottom_sum
                if difference in top_discountable:
                    return True

                difference_alt = (bottom_sum + 2*row_sums[row_cut-1]) - top_sum
                
                if row_cut == H:
                    bottom_discountable = { grid[H-1][0], grid[H-1][W-1] }
                elif W == 1:
                    bottom_discountable = { grid[row_cut-1][0], grid[H-1][0] }
                else:
                    bottom_discountable = { grid[r][c] for r in range(row_cut-1, H) for c in range(W) }

                if difference_alt in bottom_discountable:
                    return True
                else:
                    smallest_top = top_sum - max(top_discountable)
                    if smallest_top < bottom_sum:
                        continue
                    elif row_cut == 1:
                        continue
                    else:
                        break

        left_sum = 0
        right_sum = total_sum

        for col_cut in range(1, W+1):

            left_sum += col_sums[col_cut-1]
            right_sum -= col_sums[col_cut-1]

            if left_sum == right_sum:
                return True
            elif left_sum > right_sum:

                if col_cut == 1:
                    left_discountable = { grid[0][0], grid[H-1][0] }
                elif H == 1:
                    left_discountable = { grid[0][0], grid[0][col_cut-1] }
                else:
                    left_discountable = { grid[r][c] for c in range(col_cut) for r in range(H) }

                difference = left_sum - right_sum
                if difference in left_discountable:
                    return True

                difference_alt = (right_sum + 2*col_sums[col_cut-1]) - left_sum
                
                if col_cut == W:
                    right_discountable = { grid[0][W-1], grid[H-1][W-1] }
                elif H == 1:
                    right_discountable = { grid[0][col_cut-1], grid[0][W-1] }
                else:
                    right_discountable = { grid[r][c] for c in range(col_cut-1, W) for r in range(H) }

                if difference_alt in right_discountable:
                    return True
                else:
                    smallest_left = left_sum - max(left_discountable)
                    if smallest_left < right_sum:
                        continue
                    elif col_cut == 1:
                        continue
                    else:
                        break

        return False
````

## ⏱ Complexity Analysis

**Time Complexity**

```
O(H * W)
```

* Computing sums: `O(H * W)`
* Partition checks: linear scans

**Space Complexity**

```
O(H * W)
```

* Temporary sets for discountable elements

---

## 🧪 Edge Cases

1. **Already balanced grid**

```
No removal needed
```

2. **Single row or column**

```
Limited removable elements (edges only)
```

3. **No valid partition**

```
Return False
```

4. **Large imbalance**
   Early pruning stops unnecessary checks.

5. **Multiple valid answers**
   Return True on first valid partition.
