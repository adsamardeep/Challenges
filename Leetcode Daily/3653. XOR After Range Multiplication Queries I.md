# 🏷 XOR After Range Multiplication Queries

## 📜 Problem Statement  
The function processes a list of queries on an array where each query multiplies elements in a specified range with a given step size. After applying all queries, the task is to compute the XOR of all elements in the modified array. Each multiplication is performed modulo \(10^9 + 7\).

## 🔍 Example Input/Output  
Input:  
nums = [1, 2, 3, 4]  
queries = [[0, 3, 1, 2]]  

Output:  
8  

Explanation:  
After query → nums = [2, 4, 6, 8]  
XOR = 2 ^ 4 ^ 6 ^ 8 = 8  

## 🧠 Approach  
1. Define MOD = \(10^9 + 7\) to handle large values.  
2. For each query `[l, r, k, v]`:
   - Iterate from index `l` to `r` with step `k`.  
   - Multiply each selected element by `v` and take modulo.  
3. After processing all queries, compute XOR of all elements in the array.  
4. Return the final XOR result.

## 💻 Code 
```python
class Solution:
    def xorAfterQueries(self, nums: List[int], queries: List[List[int]]) -> int:
        MOD = 10**9 + 7

        for l, r, k, v in queries:
            for i in range(l, r + 1, k):
                nums[i] = (nums[i] * v) % MOD

        ans = 0
        for num in nums:
            ans ^= num

        return ans
````

## ⏱ Complexity Analysis

Time Complexity: O(q * n/k + n), where q is number of queries and each query updates roughly (r-l)/k elements.
Worst case: O(q * n).
Space Complexity: O(1), in-place updates (excluding input storage).

## 🧪 Edge Cases

* Empty nums → XOR result is 0.
* Empty queries → return XOR of original array.
* k = 1 → updates every element in range.
* Large v values → safely handled using modulo.
* Single element range (l == r) → only one update applied.
* Overlapping queries → updates accumulate correctly.
