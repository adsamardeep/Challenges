# 🔎 Find K-th Bit in N-th Binary String

## 📜 Problem Statement

Define a sequence of binary strings:

- `S1 = "0"`
- For `i > 1`:

```

Si = S(i-1) + "1" + reverse(invert(S(i-1)))

```

Where:
- `reverse(x)` → reverse string
- `invert(x)` → flip bits (`0 ↔ 1`)

Given integers `n` and `k`, return the **k-th bit** (1-indexed) in `Sn`.

---

## 🔍 Example

### Example 1:
**Input:**
```

n = 3, k = 1

```

Construct strings:

```

S1 = "0"
S2 = "011"
S3 = "0111001"

```

**Output:**
```

"0"

```

---

### Example 2:
**Input:**
```

n = 4, k = 11

```

**Output:**
```

"1"

```

---

## 🧠 Key Observations

### 1️⃣ Length of Sn

```

length = 2^n - 1

```

### 2️⃣ Structure of Sn

```

Sn = Left + "1" + Right

```

Where:
- Left = `S(n-1)`
- Middle = `'1'`
- Right = `reverse(invert(S(n-1)))`

---

## 🚀 Recursive Strategy

Let:

```

mid = (length + 1) // 2

```

### Case 1️⃣: k == mid  
Return `"1"`

### Case 2️⃣: k < mid  
It lies in left half → same as:
```

findKthBit(n-1, k)

```

### Case 3️⃣: k > mid  
It lies in right half.

Right half is:
```

reverse(invert(S(n-1)))

```

So:

1. Mirror position:
```

mirrored_index = length - k + 1

````
2. Find that bit in `S(n-1)`
3. Invert it

---

## 💻 Code 

```python
class Solution:
 def findKthBit(self, n: int, k: int) -> str:
     if n == 1:
         return '0'
     
     length = (1 << n) - 1
     mid = (length + 1) // 2
     
     if k == mid:
         return '1'
     if k < mid:
         return self.findKthBit(n - 1, k)
     
     c = self.findKthBit(n - 1, length - k + 1)
     return '1' if c == '0' else '0'
````

---

## ⏱ Complexity Analysis

### Time Complexity:

Each recursive call reduces `n` by 1.

So:

```
O(n)
```

### Space Complexity:

Recursion depth:

```
O(n)
```

---

## 🧪 Edge Cases

1. `n = 1, k = 1` → `"0"`
2. `k = mid` → always `"1"`
3. Very large `k`
4. Deep recursion (n up to ~20 safe)

---

## 🎯 Why This Is Efficient

* No full string construction
* No reverse/invert operations explicitly
* Pure recursive structure
* Uses symmetry cleverly
