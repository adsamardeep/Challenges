# 🔢 Count Binary Substrings

## 📜 Problem Statement

Given a binary string `s`, return the number of **non-empty substrings** that:

1. Contain the same number of `0`s and `1`s
2. All the `0`s and all the `1`s in the substring are grouped consecutively

---

## 🔍 Example

### Example 1:
**Input:**  
s = "00110011"

**Valid substrings:**
```

"0011"
"01"
"1100"
"10"
"0011"
"01"

```

**Output:**  
6

---

### Example 2:
**Input:**  
s = "10101"

**Output:**  
4  
("10", "01", "10", "01")

---

## 🧠 Key Insight

We don’t need to check all substrings.

Instead, think in terms of **groups of consecutive characters**.

Example:
```

"00110011"
→ groups = [2,2,2,2]

```

For each adjacent pair of groups:
```

count += min(previous_group_size, current_group_size)

````

Why?

Because a valid substring must:
- Take equal numbers from two adjacent groups
- And they must be consecutive

---

## 🧩 How Your Code Works

### Variables:

- `strk` → current streak length
- `prev` → previous streak length
- `res` → result count

### Logic:

1. Count consecutive identical characters (`strk`)
2. When character changes:
   - Save previous streak
   - Reset current streak
3. If current streak length ≤ previous streak length:
   - We found a valid substring

This works because each new character potentially forms one valid balanced substring.

---

## 💻 Code 

```python
class Solution:
    def countBinarySubstrings(self, s: str) -> int:
        res = 0
        prev = 0
        strk = 1

        for i in range(1, len(s)):
            if s[i] == s[i - 1]:
                strk += 1
            else:
                prev = strk
                strk = 1

            if strk <= prev:
                res += 1

        return res
````

---

## ⏱ Complexity Analysis

### Time Complexity:

* Single pass through string
* **O(n)**

### Space Complexity:

* Only a few variables
* **O(1)**

---

## 🧪 Edge Cases

1. `"0"` → 0
2. `"01"` → 1
3. `"1111"` → 0
4. `"000111"` → 3
5. Alternating string `"101010"` → n-1

---

## 🚀 Why This Is Optimal

* No nested loops
* No substring generation
* No extra arrays
* Just streak counting

This is the optimal solution used in interviews.
