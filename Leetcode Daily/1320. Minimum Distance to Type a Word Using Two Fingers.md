# 🏷 Minimum Distance to Type a Word Using Two Fingers

## 📜 Problem Statement  
This function computes the minimum total finger movement needed to type a given uppercase word on a keyboard grid using two fingers. At each step, one finger presses the current character while the other finger stays in place, and the goal is to minimize the total Manhattan distance traveled.

## 🔍 Example Input/Output  
Input:  
word = "CAKE"  

Output:  
3  

Explanation:  
Using two fingers optimally reduces movement. One finger can type `C` and `A`, while the other helps with `K` and `E`, giving a minimum total distance of 3.

## 🧠 Approach  
1. Model the keyboard as a 6-column grid, where each letter `A` to `Z` maps to an index from `0` to `25`.  
2. Define a helper `dist(x, y)` that returns the Manhattan distance between two letters. If either finger is in the "hovering" state (`26`), the movement cost is 0.  
3. Use dynamic programming where `dp[i][j]` stores the minimum cost after typing up to index `i`, with the **other** finger currently resting at position `j`.  
4. Initially, after typing the first character, the other finger is still unused, so set `dp[0][26] = 0`.  
5. For each next character:
   - Option 1: Keep using the same finger that typed the previous character. Add the movement cost from `prev` to current character `x`.
   - Option 2: Use the other finger to type the current character. In this case, the finger that typed `prev` becomes the new "other" finger. Add the cost from `j` to `x`.
6. Update the DP table for both choices at every step.  
7. After processing the whole word, return the minimum value in the last DP row.

## 💻 Code 
```python
# dp[i][j]=min cost after typing word[i], with the OTHER finger at j
# 26 denotes hovering
class Solution:
    def minimumDistance(self, word: str) -> int:
        def dist(x, y):
            if x==26 or y==26: return 0
            return abs(x//6-y//6)+abs(x%6-y%6)
        # setting for dp
        n=len(word)
        INF=1<<30
        dp=[[INF]*27 for _ in range(n)]
        dp[0][26]=0
        prev=ord(word[0])-65
        for i, c in enumerate(word[1:], start=1):
            x=ord(c)-65
            for j in range(27):
                dp[i][j]=min(dp[i][j], dp[i-1][j]+dist(prev, x))
                dp[i][prev]=min(dp[i][prev], dp[i-1][j]+dist(j, x))
            prev=x
        return min(dp[-1])
````

## ⏱ Complexity Analysis

Time Complexity: O(n * 27), where `n` is the length of the word. Since 27 is constant, this is effectively O(n).
Space Complexity: O(n * 27), due to the DP table.

## 🧪 Edge Cases

* Single-character word → answer is 0 because no finger movement is needed.
* Repeated letters (e.g., `"AAAA"`) → movement cost remains 0 for repeated presses at the same position.
* Short words of length 2 → DP correctly decides whether to use the same finger or the other hovering finger.
* One finger unused for many steps → hovering state `26` allows zero initial movement when that finger is first used.
* All characters different → DP explores both finger choices to ensure the global minimum is found.
