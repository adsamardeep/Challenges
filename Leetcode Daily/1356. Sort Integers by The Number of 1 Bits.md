# 🔢 Sort Integers by The Number of 1 Bits

## 📜 Problem Statement

Given an integer array `arr`, sort the integers by:

1. The number of `1`s in their binary representation  
2. If two numbers have the same number of set bits, sort them by their numeric value  

Return the sorted array.

---

## 🔍 Example

### Example 1:
**Input:**
```

arr = [0,1,2,3,4,5,6,7,8]

```

**Binary + Bit Counts:**

| Number | Binary | Set Bits |
|--------|--------|----------|
| 0      | 0      | 0        |
| 1      | 1      | 1        |
| 2      | 10     | 1        |
| 4      | 100    | 1        |
| 8      | 1000   | 1        |
| 3      | 11     | 2        |
| 5      | 101    | 2        |
| 6      | 110    | 2        |
| 7      | 111    | 3        |

**Output:**
```

[0,1,2,4,8,3,5,6,7]

````

---

## 🧠 Key Idea

Python allows sorting using a custom `key`.

We use:

```python
(x.bit_count(), x)
````

This creates a tuple for sorting:

* First priority → number of set bits
* Second priority → numeric value

Python sorts tuples lexicographically, so this works perfectly.

---

## 💻 Code 

```python
class Solution:
    def sortByBits(self, arr):
        return sorted(arr, key=lambda x: (x.bit_count(), x))
```

---

## ⏱ Complexity Analysis

Let `n = len(arr)`

### Time Complexity:

* Sorting takes **O(n log n)**
* `bit_count()` is O(1)
* Overall → **O(n log n)**

### Space Complexity:

* Sorting uses extra space
* **O(n)**

---

## 🧪 Edge Cases

1. Single element array
2. All elements same
3. Large integers
4. Already sorted input
5. Reverse sorted input

---

## 🎯 Why This Is Clean

* Uses built-in `bit_count()` (Python 3.8+)
* No manual bit counting
* No custom comparator needed
* Short and readable
