# 🔢 Partitioning Into Minimum Number Of Deci-Binary Numbers

## 📜 Problem Statement

A **deci-binary number** is a number that:
- Consists only of digits `0` and `1`
- Has no leading zeros (except the number 0)

Given a decimal string `n`, return the **minimum number of positive deci-binary numbers** needed so that they sum up to `n`.

---

## 🔍 Example

### Example 1:
**Input:**
```

n = "32"

```

**Explanation:**
```

32 = 10 + 10 + 10 + 1 + 1

```

We need **3** deci-binary numbers.

**Output:**
```

3

```

---

### Example 2:
**Input:**
```

n = "82734"

```

Largest digit = 8  

**Output:**
```

8

```

---

## 🧠 Key Insight

At each digit position:

- If the digit is `d`, we need **at least `d` deci-binary numbers** contributing `1` at that position.
- Since each deci-binary number can contribute at most `1` per digit position,
  the answer must be:

```

maximum digit in n

```

That’s it.

---

## 🚀 Why This Works

Think column-wise:

If a column has digit `7`,
you need 7 layers of `1`s stacked to build that column.

So the minimum number of deci-binary numbers required is simply:

```

max digit in the string

````

---

## 💻 Code 

```python
class Solution:
    def minPartitions(self, n: str) -> int:
        return int(max(n))
````

---

## ⏱ Complexity Analysis

Let `m = len(n)`

### Time Complexity:

* Scans string once
* **O(m)**

### Space Complexity:

* Constant extra space
* **O(1)**

---

## 🧪 Edge Cases

1. `"1"` → 1
2. `"11111"` → 1
3. `"0000"` → 0
4. Large input (10⁵ digits) → Efficient
5. Contains digit `9` → Answer is 9

---

## 🎯 Why This Is Elegant

* No simulation
* No addition logic
* No loops needed explicitly
* Just one built-in function

This is one of the cleanest greedy problems.
