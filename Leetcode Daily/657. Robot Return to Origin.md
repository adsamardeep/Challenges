# 🏷 Robot Return to Origin

## 📜 Problem Statement  
The function determines whether a robot returns to its original position after executing a sequence of moves. The robot starts at the origin (0,0), and each move shifts its position in one of four directions: up, down, left, or right. The goal is to check if the final position is the same as the starting point.

## 🔍 Example Input/Output  
Input:  
moves = "UDLR"  

Output:  
True  

## 🧠 Approach  
1. If the number of moves is odd, it's impossible to return to the origin, so return False.  
2. Initialize coordinates `x = 0` and `y = 0` to represent the starting position.  
3. Iterate through each character in the moves string.  
4. Update `y` for vertical moves:  
   - +1 for 'U' (up), -1 for 'D' (down)  
5. Update `x` for horizontal moves:  
   - +1 for 'R' (right), -1 for 'L' (left)  
6. After processing all moves, check if both `x` and `y` are zero.  
7. Return True if at origin, otherwise False.

## 💻 Code 
```python
class Solution:
    def judgeCircle(self, moves: str) -> bool:
        if len(moves) & 1: return False
        x = y = 0

        for c in moves:
            y += (c == 'U') - (c == 'D')
            x += (c == 'R') - (c == 'L')

        return not x and not y
````

## ⏱ Complexity Analysis

Time Complexity: O(n), where n is the length of the moves string.
Space Complexity: O(1), constant extra space used.

## 🧪 Edge Cases

* Empty string → robot stays at origin → True.
* Odd number of moves → cannot balance → False.
* Only one direction (e.g., "UUU") → does not return → False.
* Balanced moves but different order → still returns → True.
* Large input → still efficient due to linear scan.
