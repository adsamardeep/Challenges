## 🏷 Count Submatrices with Top-Left Origin and Sum ≤ k

## 📜 Problem Statement
Given a 2D grid of integers and an integer `k`, count the number of **submatrices starting from the top-left corner (0,0)** such that their sum is **less than or equal to `k`**. Return the total count of such valid submatrices.

## 🔍 Example Input/Output

**Example**

Input:
```
grid = [
 [1, 2],
 [3, 4]
]
k = 4
```

Output:
```
2
```

Explanation:
Valid submatrices starting at `(0,0)`:
```
[1] → sum = 1
[1,2] → sum = 3
```
Other submatrices exceed `k`.

---

## 🧠 Approach

### 1. Prefix Sum Transformation
Convert the grid into a **2D prefix sum matrix** in-place:

```
grid[i][j] = sum of rectangle from (0,0) to (i,j)
```

Formula:
```
grid[i][j] += grid[i-1][j] + grid[i][j-1] - grid[i-1][j-1]
```

### 2. Key Observation
Each cell `(i, j)` now represents the sum of a submatrix from `(0,0)` to `(i,j)`.

So, the problem reduces to:
> Count how many prefix sums are ≤ `k`.

### 3. Optimization with Early Stopping
- Track `brCol` (break column) to stop checking further columns once sum exceeds `k`.
- Since prefix sums increase monotonically across rows/columns, we can **break early** to avoid unnecessary computation.

### 4. Iteration Strategy
- First row: compute prefix sums and count valid cells.
- First column: update and count similarly.
- Remaining cells:
  - Compute prefix sum.
  - If it exceeds `k`, stop further columns for that row.

### 5. Result
Return the total count of valid prefix submatrices.

---

## 💻 Code 

```python
class Solution:
    def countSubmatrices(self, grid: List[List[int]], k: int) -> int:
        r, c=len(grid), len(grid[0])
        cnt, brCol=0, c
        if grid[0][0]>k:
            return 0
        cnt+=1
        for j in range(1, c):
            grid[0][j]+=grid[0][j-1]
            if grid[0][j]>k:
                brCol=j
                break
            cnt+=1
        for i in range(1, r):
            grid[i][0]+=grid[i-1][0]
            if grid[i][0]>k:
                break
            cnt+=1
            for j in range(1, brCol):
                grid[i][j]+=grid[i-1][j]+grid[i][j-1]-grid[i-1][j-1]
                if grid[i][j]>k:
                    brCol=j
                    break
                cnt+=1
        return cnt
```

---

## ⏱ Complexity Analysis

**Time Complexity**
```
O(r * c)
```
- Each cell is processed at most once due to early stopping.

**Space Complexity**
```
O(1)
```
- In-place modification of the grid.

---

## 🧪 Edge Cases

1. **Top-left cell exceeds k**
```
grid[0][0] > k → return 0
```

2. **Single row**
```
grid = [[1,2,3]]
```

3. **Single column**
```
grid = [[1],[2],[3]]
```

4. **All values valid**
All prefix sums ≤ `k`, count all cells.

5. **Early break scenario**
When sums exceed `k` early, loops terminate efficiently.
