# 🔁 Binary Number with Alternating Bits

## 📜 Problem Statement
Given a positive integer `n`, determine whether its binary representation has alternating bits.

A number has alternating bits if no two adjacent bits are the same.

---

## 🔍 Example Input/Output

### Example 1:
**Input:**  
n = 5  

**Binary:**  
101  

**Output:**  
True  

---

### Example 2:
**Input:**  
n = 7  

**Binary:**  
111  

**Output:**  
False  

---

### Example 3:
**Input:**  
n = 10  

**Binary:**  
1010  

**Output:**  
True  

---

## 🧠 Approach (Step-by-Step)

This solution uses a **bit manipulation trick**.

### Step 1: XOR with Right Shift
```python
x = n ^ (n >> 1)
````

If `n` has alternating bits:

* Example: `1010`
* `n >> 1` → `0101`
* XOR → `1111`

So, if bits alternate, `x` becomes a number consisting of **all 1s**.

---

### Step 2: Check if All Bits Are 1

A number consisting of all 1s has this property:

```
x & (x + 1) == 0
```

Why?

* Example: `1111` (15)
* `x + 1` → `10000`
* AND → `00000`

So if this condition holds, the number had alternating bits.

---

## 💻 Code 

```python
class Solution:
    def hasAlternatingBits(self, n: int) -> bool:
        x = n ^ (n >> 1)
        return (x & (x + 1)) == 0
```

---

## ⏱ Complexity Analysis

### Time Complexity:

* Constant bit operations
* **O(1)**

### Space Complexity:

* No extra space used
* **O(1)**

---

## 🧪 Edge Cases

1. `n = 1` → True
2. `n = 2 (10)` → True
3. `n = 3 (11)` → False
4. Single-bit numbers → Always True
5. Large integers → Works since bit operations are constant time
